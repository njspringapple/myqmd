算法复杂度与 Landau 记号

从基本定义到实际应用 —— 理解算法如何随问题规模增长而变化

⏱️ 时间复杂度

统计算法执行的基本算术运算（+ - · /）的次数。

💾 空间复杂度

统计算法运行过程中需要临时存储的数字的数量。

📈 核心问题

当数据量翻倍时，用时是翻倍还是变成四倍？还是更多？

我们通常不关心精确的操作次数，而关心复杂度（complexity）—— 即算法如何随着问题规模 n 的增长而变化。这就需要 Landau 记号来形式化描述。

📐 Landau 记号概述

Landau 记号是分析算法复杂度的核心工具，它允许我们忽略常数因子和低阶项，只关注增长的"量级"。两个最重要的记号是大 O（Big O）和小 o（Little o）。

大 O 记号 — O(·)

上界 至多同阶
若 limsupn→∞ |an / bn| < ∞，则 an = O(bn)

直觉：an 的增长速度不会超过 bn（至多差一个常数倍）。

重要特例：

• an = O(1) ⟹ an 有界

• 1000n = O(n)，忽略常数因子

小 o 记号 — o(·)

严格慢于 可忽略
若 limn→∞ |an / bn| = 0，则 an = o(bn)

直觉：an 的增长速度严格慢于 bn，比值趋于零。

重要特例：

• an = o(1) ⟹ an 收敛到 0

• n² = o(n³)，因为 n²/n³ = 1/n → 0

⚖️ 大 O 与 小 o 的对比

属性 大 O（Big O） 小 o（Little o）
定义 limsup |an/bn| < ∞ lim |an/bn| = 0
含义 an 增长至多像 bn an 增长严格慢于 bn
类比 ≤（不超过，忽略常数） <（严格小于）
蕴含关系 an = o(bn) ⟹ an = O(bn)，反之不成立
例：n² vs n² n² = O(n²) ✅ n² = o(n²) ❌
例：n² vs n³ n² = O(n³) ✅ n² = o(n³) ✅

⚠️ 实践中的注意事项

Landau 记号忽略常数因子。例如 an = 1000n = O(n) 而 bn = 0.1n² = O(n²)。虽然 bn 的复杂度"更差"，但在 n < 10000 时，an 实际上更大！

结论：复杂度更差的算法在实际中可能因为常数、实现方式、硬件缓存等因素而更快。复杂度分析描述的是渐近行为。

🎯 主导项化简

核心思想：一个表达式的复杂度由其增长最快的项（主导项）决定。

化简规则

10⁶ + 100n + 2n² = O(n²) = o(n³)

当 n → ∞ 时，n² 项的增长速度远超其他所有项，因此它"主导"了整个表达式。

1/n² + 1/n³ = O(1/n²) = o(1/n)

对于趋于 0 的序列，1/n² 衰减最慢（值最大），是主导项。

🧲 复杂度吸收

当多个复杂度项相加时，高阶项会"吸收"低阶项。这是化简复杂度表达式的核心技巧。

吸收原理

O(f) + O(g) = O(max{f, g})

如果 f(n) ≤ g(n)（对足够大的 n），则 O(f) ⊆ O(g)，于是 O(f) + O(g) = O(g)。

实际案例：组合方差算法的复杂度化简

1

各步骤复杂度：colMeans O(nd)，循环体 O(nd + d²)，循环 n 次 O(n²d + nd²)，最终乘法 O(d²)

2

3

吸收：O(nd) ⊆ O(n²d)，O(d²) ⊆ O(nd²)（当 n ≥ 1）

4

最终结果：O(n²d + nd²)

🔗 传递性规则

O 的传递性

an = O(bn)，bn = O(cn)
⟹ an = O(cn)

O 到 o 的传递

an = O(bn)，bn = o(cn)
⟹ an = o(cn)

应用实例（练习 1.2）

已知 an = O(en + en²)。

情况 1（en → ∞）：此时 en² 主导，(en + en²)/en² = 1/en + 1 → 1，因此 en + en² = O(en²)，传递可得 an = O(en²)。

情况 2（en → 0）：此时 en 主导，(en + en²)/en = 1 + en → 1，因此 en + en² = O(en)，传递可得 an = O(en)。

⏱️💾 时间与空间复杂度

⏱️ 时间复杂度

统计基本算术运算（+ - × ÷）的总次数，用 Landau 记号表示其关于输入大小 n 的增长阶。

💡 关键洞察

利用数学性质（如迹的线性性）可以避免不必要的中间计算。先计算 A+B（O(n²)）再取迹（O(n)）的"笨"方法总复杂度为 O(n²)，而直接 tr(A)+tr(B) 只需 O(n)。选择正确的计算顺序至关重要！

📈 协方差矩阵

什么是协方差矩阵？

给定 d 维随机向量 X，其协方差矩阵 Σ 是一个 d×d 矩阵，Σij = Cov(Xi, Xj)。对角线是各分量的方差，非对角线是两两之间的协方差。

样本估计公式

Σ̂ = (1/n) Σi=1n (Xi - μ̂)(Xi - μ̂)T
μ̂ = (1/n) Σi=1n Xi

其中 Xi ∈ ℝd 是 n 个 d 维观测向量。

🏦 投资组合应用

场景：你买了 d 支 ETF，持有比例为 β = (β₁, …, βd)T。

组合方差：σ²(β) = βT Σ β —— 衡量投资组合的风险。

估计：用 n 天的历史数据估计 Σ̂，进而得到 σ̂²(β) = βT Σ̂ β。

🧮 组合方差算法的复杂度分析

算法伪代码

function sigma_2(X[n×d], beta[d]):
  Sigma ← 0d×d               // 初始化 d×d 零矩阵
  mu ← colMeans(X)       // 计算各列均值，O(nd)
  for i = 1 to n:
    v ← X[i] - mu           // d 次减法
    Sigma += v · vT        // 外积 d² + 矩阵加法 d²
  return betaT · Sigma · beta / n

⚠️ 注意：代码中 colMeans 在循环内部调用！

原始 R 代码在每次循环中重新计算 colMeans(X)，每次耗费 O(nd)。循环 n 次后，仅这部分就已经是 O(n²d)。这是一个可以优化的低效设计（将 colMeans 提到循环外只需 O(nd) 一次）。

时间复杂度分析

1

colMeans(X)：对 d 列各做 n-1 次加法 → O(nd)

2

单次循环体：colMeans O(nd) + 减法 O(d) + 外积 O(d²) + 矩阵加法 O(d²) → O(nd + d²)

3

n 次循环：n × O(nd + d²) → O(n²d + nd²)

4

βTΣβ：矩阵向量乘 O(d²) + 内积 O(d) → O(d²)

🧲 吸收过程

步骤 1 的 O(nd) 被步骤 3 的 O(n²d) 吸收（因为 nd ≤ n²d）。
步骤 4 的 O(d²) 被步骤 3 的 O(nd²) 吸收（因为 d² ≤ nd²）。

总时间复杂度：O(n²d + nd²)

X

输入矩阵 n × d → O(nd)

β

参数向量 d × 1 → O(d)

Σ̂

协方差矩阵 d × d → O(d²)

总空间复杂度：O(nd + d²)

其中 O(d) 被 O(d²) 吸收。

✏️ 练习 1.1 — 找 O 和 o 的上界

对给定序列 an，找更简单的 bn 和 cn，使得 an = O(bn) 且 an = o(cn)。

(a) an = 100n-1 + 20n0.5 + 5n1.5

主导项：n1.5（增长最快）

→

取 bn = n1.5：
an/bn = 100n-2.5 + 20n-1 + 5 → 5，极限有限 ✅ → an = O(n1.5)

→

取 cn = n2：
an/cn = 100n-3 + 20n-1.5 + 5n-0.5 → 0 ✅ → an = o(n2)

结果：an = O(n1.5) = o(n²)

(b) an = 2log(n)/n + log(2)/n² + log(1+1/n)

关键提示：log(1+1/n) ≤ 1/n

→

取 bn = log(n)/n：三项分别与 bn 的比值为 2（常数）、→0、→0。
因此 an = O(log(n)/n)。

→

取 cn = 1：因为 bn = log(n)/n → 0 = o(1)，传递性可得 an = o(1)。

结果：an = O(log(n)/n) = o(1)
经典结论：序列极限为 0 ⟺ o(1)

✏️ 练习 1.2 — 主导项随极限变化

设 an = O(en + en²)，证明不同极限行为下的化简。

情况 1：en → ∞

en² 主导

(en + en²) / en² = 1/en + 1 → 1
⟹ en + en² = O(en²)
⟹ an = O(en²)

情况 2：en → 0

en 主导

(en + en²) / en = 1 + en → 1
⟹ en + en² = O(en)
⟹ an = O(en)

💡 一句话总结

en 很大时 en² 主导（平方比线性增长快），en 很小时 en 主导（平方比线性衰减快）。谁"更显眼"谁就是主导项。
