2.1 Git 到底是什么？ | Was ist Git wirklich?

Note定义 (Definition)

Git 是一个内容寻址文件系统 (content-addressable filesystem)，上面搭建了一个版本控制界面 (VCS user interface)。

Git 由 Linus Torvalds 为 Linux 内核开发而创建。Git 中所有数据通过 SHA-1 哈希值标识。它的设计理念是简单的构建块，组合起来却非常强大。

Git 的命令分为两个层次。瓷器命令 (Porcelain)是面向用户的高级命令，如 checkout、branch、remote 等，书中前九章主要讨论的就是这些。管道命令 (Plumbing) 是底层命令，最初被设计为可以按 UNIX 风格链式调用或在脚本中使用的工具集，它们为我们揭示了 Git 的内部运作机制。

2.3 哈希函数 | Hash-Funktionen

哈希函数 (h: K S) 从给定的 (K) 类型数据生成一个（极大概率）唯一的指纹，映射到集合 (S)。由于 (|K| |S|)，碰撞在理论上是可能的，但在实践中极为罕见。

哈希函数的三个重要性质：确定性 (Deterministisch)——相同输入始终产生相同输出；单向性 (Einweg)——无法从哈希值反推原始输入；抗碰撞 (Kollisionsresistent)——实际上不可能找到两个不同的输入具有相同哈希值。

2.4 Git 如何计算哈希 | Wie Git Hashes berechnet

Git 使用 SHA-1 (Secure Hash Algorithm 1)，从给定的字节序列生成 160 位（40 个十六进制字符）的哈希值。虽然 SHA-1 在密码学上已不安全，但对于版本控制来说是足够的。

Git 计算哈希的步骤是：先将 Header 与内容连接在一起（格式为 <类型> <大小>\0<内容>，其中 \0 是空字节分隔符），然后对整体计算 SHA-1 哈希。

# 手动计算
❱ echo -n "blob 16\0what is up, doc?" | sha1sum --quiet
bd9dbf5aae1a3862dd1526723246b20206e5fc37

# 使用 Git 自带命令
❱ echo -n "what is up, doc?" | git hash-object --stdin
bd9dbf5aae1a3862dd1526723246b20206e5fc37

2.8 为什么不浪费空间？| Warum ist das nicht verschwenderisch?

通过哈希实现自动去重：因为对象按其内容的哈希值存储，所以如果两个不同 Commit 中同一个文件没有改变，它们指向的是同一个 Blob 对象。两个不同 Commit 的 Tree 中，未变化的子目录也指向相同的 Tree 对象。这意味着 Git 只存储每个独特文件内容的一份副本。

2.9 git diff 的工作原理 | Funktionsweise von git diff

Diff 是按需计算的，不是预先存储的。当你运行 git diff 时，Git 会比较两个 Commit 的 Tree 对象，找出差异并动态生成差异输出。diff 输出中的 -22,10 表示旧版本从第 22 行起 10 行，+22,7 表示新版本从第 22 行起 7 行。

4.2.1 Rebase：线性化历史 | Geschichte linearisieren

Rebase 将 feature 分支的 Commit “移植”到 main 分支的末端。git rebase main（在 feature 分支上执行）的结果是创建了 F’ 和 G’——它们具有与原 F、G 相同的更改，但有全新的哈希值，因为它们的 parent 改变了。结果是一条干净的线性历史。

4.3 黄金规则 (Goldene Regel)

永远不要 rebase 已经推送 (push) 过的 Commit！ 因为 rebase 会重写 Commit 哈希，这将导致与其他协作者的仓库不一致。

4.3.1 Merge：连接历史 | Historien verbinden

Merge 创建一个新的 Merge Commit M，它有两个 parent（分别指向两条分支的最新 Commit）。两条历史都完整保留，没有 Commit 被重写。这比 Rebase 更安全。

Merge Commit 与普通 Commit 的唯一区别在于它有多个 parent 行：

❱ git cat-file -p 546e070
tree d18f261f6a6...
parent 86e8bb5a6fd...    # ← 第一个 parent (main)
parent fd61b5787f3...    # ← 第二个 parent (feature)
author Thomas Lemberger <...>
committer Thomas Lemberger <...>

Merge branch 'support-constant-expressions-in-typedefs' into 'main'

4.5 合并冲突 | Merge-Konflikte

当 Git 无法自动决定采用哪个版本时，就会产生冲突。冲突的文件中会出现冲突标记：

<<<<<<< HEAD
code from current branch
=======
code from incoming branch
>>>>>>> feature

解决步骤：(1) 删除冲突标记；(2) 保留需要的代码；(3) 提交。
