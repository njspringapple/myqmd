1.1 数值问题是什么？

Was ist ein numerisches Problem?

一个 数值问题 的解是一个或多个数字。例如：求积分的值、求解线性方程组、求函数逼近中的系数。一般来说，给定一个带输入 \(x\) 的问题 \(f\)，我们需要计算它的解 \(f(x)\)。

Ein numerisches Problem hat als Lösung eine oder mehrere Zahlen. Allgemein: Gegeben ein Problem \(f\) mit Eingabe \(x\), berechne die Lösung \(f(x)\).

1.2 根本困难：有限 vs 无限

Grundlegende Schwierigkeit: Endlich vs. Unendlich

计算机 几乎从来都不是精确的。它只能表示 有限位数 的 有限多个 数字，并且只能进行 有限次 计算。

2.1 灾难性消去

Katastrophale Auslöschung

当两个 几乎相等的大数相减 时，结果的有效数字会大量丢失，误差被放大到离谱的程度。这就是灾难性消去。

经典例子：方差的两种等价公式

在代数上，以下两个公式完全等价：

不稳定版本（容易灾难性消去）：

\[ \text{Var}(x) = \frac{1}{n}\sum_{i=1}^{n} x_i^2 - \left(\frac{1}{n}\sum_{i=1}^{n} x_i\right)^2 \]

稳定版本（推荐）：

\[ \text{Var}(x) = \frac{1}{n}\sum_{i=1}^{n} (x_i - \bar{x})^2 \]

为什么不稳定？当数据整体很大（比如都在 \(10^9\) 附近），但波动很小（差异只有十几）时，\(\frac{1}{n}\sum x_i^2\) 和 \(\bar{x}^2\) 都是 \(10^{18}\) 量级，它们的差（方差）却只有 \(10^1\) 量级。两大数相减会把高位相同部分消掉，剩下的低位受舍入影响极大。

2.2 加法不满足结合律

在数学上，加法满足结合律 \((a + b) + c = a + (b + c)\)。但在浮点数运算中，这不一定成立。

为什么第二种顺序得到 0？因为在浮点数里，相对于 \(10^{30}\) 来说，\(1\) 太小了。计算 \((-10^{30}) + 1\) 时，\(1\) 被舍入”吃掉”，结果仍然是 \(-10^{30}\)。

Warning⚠️ 核心教训 | Kernlektion

代数恒等式 ≠ 数值稳定。编写数值算法时，必须考虑：运算顺序、中间值大小、以及可能的精度损失。建议使用”先中心化再计算”的策略，或使用数值稳定算法（如 Welford 在线算法计算方差）。

Algebraische Identität ≠ Numerische Stabilität. Bei numerischen Algorithmen müssen Reihenfolge, Zwischenwertgrößen und möglicher Präzisionsverlust berücksichtigt werden.

3.1 算法的定义与分类

Definition und Klassifikation von Algorithmen

算法是一种过程（由若干步骤组成），对输入 \(x\) 计算一个（可能的）输出/解 \(\tilde{f}(x)\)。

Ein Algorithmus ist ein Verfahren (bestehend aus mehreren Schritten), das für eine Eingabe \(x\) eine (mögliche) Ausgabe/Lösung \(\tilde{f}(x)\) berechnet.

3.3 时间开销与空间开销

Zeit- und Speicheraufwand

计算机执行算法时，会用 基本算术运算 \(f_i \in \{+, -, \cdot, /\}\)。

算法的 时间开销（Zeitaufwand） 是这些基本运算的次数——代价越高，运行时间越长。算法的 空间开销（Speicheraufwand） 大致是需要存储的（中间）数字的数量——代价越高，需要更多内存。

3.5 详细拆解：循环算法的逐步计数

Detaillierte Aufschlüsselung: Operationen pro Iteration

很多算法的核心是一个循环。让我们学会逐步计数每次迭代中的基本运算。

考虑一个典型的循环结构：对 \(i = 1, \ldots, d\)，执行以下操作——计算标量差值 \((x_i - y_i)\)（1 次减法）、将差值乘以向量 \(\mathbf{a}_i\)（\(n\) 次乘法）、累加到结果向量 \(\mathbf{f}\)（\(n\) 次加法）。

因此每次迭代的总开销为 \(1 + n + n = 2n + 1\) 次运算。做 \(d\) 轮后，总开销为 \(d \cdot (2n + 1) = 2nd + d\) 次基本运算。

4.1 大 \(O\) 与小 \(o\)：定义

Definition von \(O\) und \(o\)

对于两个数列 \(a_n\) 和 \(b_n\)（\(b_n > 0\)）：

小 \(o\)（严格小一阶）： \(a_n = o(b_n)\) 当且仅当

\[ \lim_{n \to \infty} \frac{a_n}{b_n} = 0 \]

直觉：\(a_n\) 和 \(b_n\) 比起来 可以忽略不计。

大 \(O\)（至多同阶）： \(a_n = O(b_n)\) 当且仅当

\[ \limsup_{n \to \infty} \left|\frac{a_n}{b_n}\right| < \infty \]

等价地：存在常数 \(C > 0\) 和 \(N_0\)，使得对所有 \(n \geq N_0\)，\(|a_n| \leq C \cdot |b_n|\)。

直觉：\(a_n\) 增长速度不超过 \(b_n\) 的常数倍。

并且：\(a_n = o(b_n) \Rightarrow a_n = O(b_n)\)（反过来不一定成立）。

4.4 Landau 记号的计算规则

Rechenregeln der Landau-Notation

设 \(f_n\)、\(g_n\) 是两个数列，\(a_n\)、\(b_n\) 是参考序列。

规则 1（同阶相运算）： 若 \(f_n = O(a_n)\) 且 \(g_n = O(b_n)\)，则 \(f_n + g_n = O(a_n + b_n)\) 且 \(f_n \cdot g_n = O(a_n \cdot b_n)\)。

规则 2（可忽略因子的传递）： 若 \(f_n = O(a_n)\) 且 \(g_n = o(b_n)\)，则 \(f_n \cdot g_n = o(a_n \cdot b_n)\)。直觉：一个因子”可忽略小”，乘出来仍然”可忽略小”。

规则 3（吸收较小项）： 若 \(f_n = O(a_n + b_n)\) 且 \(a_n = O(b_n)\)（\(a_n\) 被 \(b_n\) 支配），则 \(f_n = O(b_n)\)。直觉：\(a_n\) 被 \(b_n\) 吸收。

传递性： 若 \(a_n = O(b_n)\) 且 \(b_n = O(c_n)\)，则 \(a_n = O(c_n)\)。

重要特例： \(a_n = O(1)\) 意味着 \(a_n\) 有界；\(a_n = o(1)\) 意味着 \(a_n\) 收敛到 0。

通用结论： \(a_n = O(b_n)\) 且 \(b_n = o(c_n) \Rightarrow a_n = o(c_n)\)。

5.1 统计运算的复杂度

Komplexität statistischer Operationen

5.1.1 均值 | Mittelwert

给定 \(X_1, \ldots, X_n \in \mathbb{R}\)，计算 \(\bar{X} = \frac{1}{n}\sum_{i=1}^{n} X_i\) 需要 \(n - 1\) 次加法和 \(1\) 次除法，因此复杂度为 \(O(n)\)。

5.1.2 距离矩阵 | Distanzmatrix

\(D \in \mathbb{R}^{n \times n}\)，其中 \(D_{ij} = |X_i - X_j|\)。一共有 \(n^2\) 个元素，每个需要一次减法和取绝对值，因此复杂度为 \(O(n^2)\)。

5.1.3 矩阵乘法 | Matrixmultiplikation

\(A \in \mathbb{R}^{m \times n}\)、\(B \in \mathbb{R}^{n \times \ell}\)，计算 \(AB \in \mathbb{R}^{m \times \ell}\)。标准算法中，\((AB)_{ij} = \sum_{k=1}^{n} A_{ik} B_{kj}\)。每个输出元素需要 \(n\) 次乘法和 \(n - 1\) 次加法，即 \(O(n)\)。共有 \(m\ell\) 个输出元素，因此总复杂度为 \(O(mn\ell)\)。

5.1.4 矩阵的迹 | Spur einer Matrix

\(\text{tr}(A) = \sum_{i=1}^{n} A_{ii}\) 只需对 \(n\) 个对角元素求和，复杂度为 \(O(n)\)。

巧妙的地方在于：\(\text{tr}(A + B) = \text{tr}(A) + \text{tr}(B)\)，每个迹各需 \(O(n)\)，最后相加 \(O(1)\)，总共 \(O(n)\)。如果你先显式计算 \(A + B\)（\(O(n^2)\)），再取迹，反而更慢！

类似地，\(\text{tr}(cA) = c \cdot \text{tr}(A)\)，复杂度为 \(O(n) + O(1) = O(n)\)。如果先计算 \(cA\)（\(O(n^2)\)），再取迹，同样浪费了算力。

5.2 斐波那契数列：迭代 vs 递归

斐波那契数列 \(F_0 = 0, F_1 = 1, F_n = F_{n-1} + F_{n-2}\) 是复杂度分析的经典教学案例。两种实现方式的效率差异极其 戏剧性。

# === 迭代版本 | Iterative Version ===
# 时间：O(n)，空间：O(n)
fib_iterative <- function(n) {
  if (n <= 1) return(n)
  x <- numeric(n + 1)
  x[2] <- 1  # x[1]=F(0)=0, x[2]=F(1)=1
  for (i in 3:(n + 1)) {
    x[i] <- x[i - 1] + x[i - 2]
  }
  return(x[n + 1])
}

# === 朴素递归版本 | Naive rekursive Version ===
# 时间：O(2^n)，空间：O(n)（调用栈深度）
fib_recursive <- function(n) {
  if (n <= 1) return(n)
  return(fib_recursive(n - 1) + fib_recursive(n - 2))
}

6.2 练习 2：主导项随条件变化

Übung 2: Dominanter Term ändert sich mit Bedingungen

设 \(e_n\) 是一个数列，且 \(a_n = O(e_n + e_n^2)\)。证明：

  1. 若 \(e_n \to \infty\)，则 \(a_n = O(e_n^2)\)。

情况 1： \(e_n \to \infty\)。比较 \(e_n + e_n^2\) 与 \(e_n^2\)：

直觉：\(e_n\) 很大时，\(e_n^2\) 远大于 \(e_n\)，\(e_n^2\) 主导。

情况 2： \(e_n \to 0\)。比较 \(e_n + e_n^2\) 与 \(e_n\)：

\[ \frac{e_n + e_n^2}{e_n} = 1 + e_n \xrightarrow{n \to \infty} 1 + 0 = 1 \]

因此 \(e_n + e_n^2 = O(e_n)\)。由传递性得 \(a_n = O(e_n)\)。

直觉：\(e_n\) 很小时，\(e_n^2\) 比 \(e_n\) 更小（例如 \(0.01^2 = 0.0001\)），\(e_n\) 主导。

Übung 3: Komplexität der Kovarianzmatrixschätzung

设 \(X_1, \ldots, X_n\) 是 \(\mathbb{R}^d\) 中的独立同分布随机向量，协方差矩阵 \(\Sigma\) 的估计为：

\[ \hat{\Sigma} = \frac{1}{n}\sum_{i=1}^{n}(X_i - \hat{\mu})(X_i - \hat{\mu})^T, \quad \hat{\mu} = \frac{1}{n}\sum_{i=1}^{n} X_i \]

线性组合 \(\beta^T X\) 的方差估计为 \(\hat{\sigma}^2(\beta) = \beta^T \hat{\Sigma} \beta\)。

下面是对应的 R 算法：

sigma_2 <- function(X, beta) {
  n <- nrow(X)
  d <- ncol(X)
  
  Sigma <- matrix(0, d, d)
  
  for (i in 1:n) {
    # 每次循环都重新计算 colMeans(X)!
    Sigma <- Sigma + (X[i,] - colMeans(X)) %*% t(X[i,] - colMeans(X))
  }
  
  as.numeric(t(beta) %*% Sigma %*% beta / n)
}

(a) 分析该算法的时间复杂度和空间复杂度。

Tip🔍 点击查看解答 | Lösung anzeigen

空间复杂度：

\(X\) 需要 \(O(nd)\)，\(\beta\) 需要 \(O(d)\)，\(\hat{\Sigma}\) 需要 \(O(d^2)\)。总空间为 \(O(nd + d^2)\)。

时间复杂度（逐步分析）：

colMeans(X) 对每一列做 \(n - 1\) 次加法和 \(1\) 次除法，共 \(d\) 列，因此单次调用需要 \(O(nd)\)。

在循环的第 \(i\) 次迭代中：调用 colMeans(X) 需要 \(O(nd)\)；计算 \(X[i,] - \text{colMeans}(X)\) 需要 \(d\) 次减法，即 \(O(d)\)；计算外积 \((X_i - \hat{\mu})(X_i - \hat{\mu})^T\) 需要 \(d^2\) 次乘法，即 \(O(d^2)\)；矩阵加法 Sigma + ... 需要 \(d^2\) 次加法，即 \(O(d^2)\)。每次迭代总共 \(O(nd + d^2)\)。

\(n\) 次迭代的总时间为 \(n \cdot O(nd + d^2) = O(n^2 d + nd^2)\)。

最后计算 \(\beta^T \hat{\Sigma} \beta\)：\(\beta^T \hat{\Sigma}\) 是 \(1 \times d\) 乘 \(d \times d\)，需要 \(O(d^2)\)；结果再与 \(\beta\) 做内积需要 \(O(d)\)。这一步为 \(O(d^2)\)，被循环部分吸收。

总时间复杂度：\(O(n^2 d + nd^2)\)

注意：此算法的瓶颈在于 每次循环都重新计算 colMeans(X)。如果预先计算一次均值（\(O(nd)\)），每次迭代只需 \(O(d^2)\)，总时间可降至 \(O(nd + nd^2) = O(nd^2)\)。
