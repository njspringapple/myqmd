矩阵分解的思想完全相同：把一个”复杂”的矩阵 \(\mathbf{A}\) 拆成几个”简单”矩阵的乘积。

什么叫”简单”？在数值线性代数中，有三类矩阵特别好处理：三角矩阵（求解线性方程组只需 \(O(n^2)\)）、正交矩阵（数值稳定，不放大误差）、对角矩阵（运算几乎是平凡的）。

2.2 一个实际问题

假设你是一名工程师，需要模拟桥梁在不同载荷下的变形。物理模型离散化后得到一个线性方程组 \(\mathbf{Ax} = \mathbf{b}\)，其中 \(\mathbf{A}\) 是描述桥梁结构的刚度矩阵（不变），\(\mathbf{b}\) 是外力（每种载荷不同）。

你需要对 1000 种不同的载荷 \(\mathbf{b}_1, \mathbf{b}_2, \ldots, \mathbf{b}_{1000}\) 分别求解。

  • 任何矩阵加上自己的转置，结果一定是对称矩阵

  • LU 分解就是把一个矩阵拆成两个矩阵的乘积，A = LU，其中 L 是下三角矩阵（对角线以上都是零），U 是上三角矩阵（对角线以下都是零）

library(Matrix)

# 演示：重复求解 vs 一次分解
set.seed(42)
# 构造对称且正定的方阵
n <- 200
A <- matrix(rnorm(n^2), n, n)
A <- A + t(A) + n * diag(n)

J_values <- c(1, 10, 50, 100, 500, 1000)
time_naive <- numeric(length(J_values))
time_lu <- numeric(length(J_values))

for (idx in seq_along(J_values)) {
  J <- J_values[idx]
  B <- matrix(rnorm(n * J), n, J)

  # 方法1：每次都"从头求解"
  time_naive[idx] <- system.time({
    for (j in 1:J) solve(A, B[, j])     # O(n^3) 循环做，慢
  })["elapsed"]

  # 方法2：一次 LU 分解，重复使用
  time_lu[idx] <- system.time({
    A_sparse <- Matrix(A)
    # @x：存放 L 和 U 矩阵的数据，压缩在一个长度 40000（200×200）的向量里。
    lu_decomp <- lu(A_sparse)    # O(n^3)  只计算一次 O(n^3)
    # lu_decomp$L   lu_decomp$U   存储上下 三角矩阵
    for (j in 1:J) solve(lu_decomp, B[, j])     # O(n^2) 循环做
  })["elapsed"]
}

plot(J_values, time_naive, type = "b", pch = 19, col = "tomato", lwd = 2,
     xlab = "右端向量个数 J", ylab = "计算时间 (秒)",
     main = "重复求解 vs 一次分解：分解的威力",
     ylim = c(0, max(time_naive) * 1.1))
lines(J_values, time_lu, type = "b", pch = 17, col = "steelblue", lwd = 2)
grid()
legend("topleft", c("每次重新求解 O(Jn³)", "一次分解 + 重复回代 O(n³ + Jn²)"),
       col = c("tomato", "steelblue"), pch = c(19, 17), lwd = 2)

关键洞察：如果你需要对同一个 \(\mathbf{A}\) 求解多个不同的 \(\mathbf{b}\)，一次分解 \(\mathbf{A} = \mathbf{LU}\) 之后，每个新的 \(\mathbf{b}\) 只需要 \(O(n^2)\) 的回代操作，而不是 \(O(n^3)\) 的完整求解。

3 矩阵运算的复杂度

在讨论矩阵分解之前，我们先建立对基本矩阵运算复杂度的直觉。

3.1 矩阵加法

两个矩阵 \(\mathbf{A}, \mathbf{B} \in \mathbb{R}^{n \times m}\) 相加，需要对每个元素做一次加法：

\[ \mathbf{A} + \mathbf{B} = \begin{pmatrix} a_{11} + b_{11} & \cdots & a_{1m} + b_{1m} \\ \vdots & & \vdots \\ a_{n1} + b_{n1} & \cdots & a_{nm} + b_{nm} \end{pmatrix} \]

共 \(nm\) 个元素，因此复杂度为 \(O(nm)\)。

# 验证矩阵加法的线性复杂度
sizes <- c(100, 200, 500, 1000, 2000)
times_add <- numeric(length(sizes))

for (i in seq_along(sizes)) {
  n_s <- sizes[i]
  A_s <- matrix(rnorm(n_s^2), n_s, n_s)
  B_s <- matrix(rnorm(n_s^2), n_s, n_s)
  times_add[i] <- median(replicate(5, system.time(A_s + B_s)["elapsed"]))
}

plot(sizes^2, times_add, type = "b", pch = 19, col = "steelblue", lwd = 2,
     xlab = "矩阵元素个数 n²", ylab = "计算时间 (秒)",
     main = "矩阵加法：时间与元素个数成正比")
grid()
# 拟合线性关系

  abline(fit_add, col = "tomato", lty = 2, lwd = 2)

3.2 矩阵乘法

两个矩阵 \(\mathbf{A} \in \mathbb{R}^{n \times m}\) 和 \(\mathbf{B} \in \mathbb{R}^{m \times k}\) 相乘，结果矩阵的每个元素需要 \(m\) 次乘法和 \(m-1\) 次加法：

\[ (\mathbf{AB})_{ij} = \sum_{\ell=1}^{m} a_{i\ell} b_{\ell j} \]

结果矩阵有 \(nk\) 个元素，每个需要 \(O(m)\) 次运算，因此总复杂度为 \(O(nmk)\)。

对于方阵（\(n = m = k\)），复杂度为 \(O(n^3)\)。

4 线性方程组：为什么不求逆？

4.1 问题设定

我们要求解线性方程组 \(\mathbf{Ax} = \mathbf{b}\)，其中 \(\mathbf{A} \in \mathbb{R}^{n \times n}\) 可逆，\(\mathbf{b} \in \mathbb{R}^n\)。

数学上，解为 \(\mathbf{x} = \mathbf{A}^{-1}\mathbf{b}\)。但在数值计算中，永远不要显式计算 \(\mathbf{A}^{-1}\)！

三个理由不要求逆：

  1. 更慢：计算 \(\mathbf{A}^{-1}\) 需要 \(O(n^3)\)，再乘以 \(\mathbf{b}\) 还需要 \(O(n^2)\)。直接求解也是 \(O(n^3)\)，但常数更小。
  2. 不更准：求逆并不能提高精度，有时反而更差。
  3. 浪费内存：\(\mathbf{A}^{-1}\) 通常是稠密矩阵（即使 \(\mathbf{A}\) 是稀疏的）。

R 中的规则：用 solve(A, b) 而不是 solve(A) %*% b。

4.2 三角方程组的高效求解

如果 \(\mathbf{A}\) 是三角矩阵，求解特别快。

上三角回代：对于上三角系统 \(\mathbf{Ux} = \mathbf{c}\)：

\[ x_n = \frac{c_n}{u_{nn}}, \qquad x_i = \frac{1}{u_{ii}}\left(c_i - \sum_{j=i+1}^{n} u_{ij} x_j\right), \quad i = n-1, \ldots, 1 \]

# 手写回代算法
back_substitution <- function(U, c) {
  n <- nrow(U)
  x <- numeric(n)
  x[n] <- c[n] / U[n, n]
  for (i in (n-1):1) {
    x[i] <- (c[i] - sum(U[i, (i+1):n] * x[(i+1):n])) / U[i, i]
  }
  x
}

# 手写前向替换算法
forward_substitution <- function(L, b) {
  n <- nrow(L)
  y <- numeric(n)
  y[1] <- b[1] / L[1, 1]
  for (i in 2:n) {
    y[i] <- (b[i] - sum(L[i, 1:(i-1)] * y[1:(i-1)])) / L[i, i]
  }
  y
}

5 LU 分解

5.1 直觉：高斯消元的矩阵语言

你在线性代数课上学过高斯消元法：对增广矩阵 \((\mathbf{A} \mid \mathbf{b})\) 做行操作，化为行阶梯形 \((\mathbf{U} \mid \mathbf{c})\)。

LU 分解的核心思想是：把高斯消元的每一步操作记录下来，存储在一个下三角矩阵 \(\mathbf{L}\) 中。

\[ \mathbf{A} = \mathbf{L}\mathbf{U} \]

其中 \(\mathbf{L}\) 是下三角矩阵（存储消元步骤），\(\mathbf{U}\) 是上三角矩阵（消元结果）。

5.2 消元矩阵

高斯消元的每一步可以用一个消元矩阵 \(\mathbf{L}_k\) 来表示。在第 \(k\) 步，我们消去第 \(k\) 列中对角线以下的所有元素。

消元矩阵的形式为：

\[ \mathbf{L}_k = \begin{pmatrix} 1 & & & & \\ & \ddots & & & \\ & & 1 & & \\ & & -l_{(k+1),k} & 1 & \\ & & \vdots & & \ddots \\ & & -l_{n,k} & & & 1 \end{pmatrix} \]

其中 \(l_{ik} = m_{ik} / m_{kk}\) 是消元因子（\(m_{kk}\) 是当前的主元）。

5.3 从消元矩阵到 L

5.6.2 部分选主元

策略：在第 \(k\) 步，选择第 \(k\) 列中绝对值最大的元素作为主元（交换行）。

# 手动实现带选主元的 LU 分解
lu_partial_pivot <- function(A) {
  n <- nrow(A)
  L <- diag(n)
  U <- A
  P <- diag(n)  # 置换矩阵

  for (k in 1:(n-1)) {
    # 选主元：在第 k 列第 k 行以下找最大元素
    max_idx <- which.max(abs(U[k:n, k])) + k - 1

    if (max_idx != k) {
      # 交换 U 的行
      U[c(k, max_idx), ] <- U[c(max_idx, k), ]
      # 交换 P 的行
      P[c(k, max_idx), ] <- P[c(max_idx, k), ]
      # 交换 L 中已计算的部分
      if (k > 1) {
        L[c(k, max_idx), 1:(k-1)] <- L[c(max_idx, k), 1:(k-1)]
      }
    }

    # 消元
    for (i in (k+1):n) {
      L[i, k] <- U[i, k] / U[k, k]
      U[i, ] <- U[i, ] - L[i, k] * U[k, ]
    }
  }

  list(L = L, U = U, P = P)
}

6.1 SPD 矩阵：统计学中最重要的矩阵类

定义：矩阵 \(\mathbf{A} \in \mathbb{R}^{n \times n}\) 称为SPD（对称正定）矩阵，如果

  1. \(\mathbf{A} = \mathbf{A}^T\)（对称）
  2. \(\mathbf{x}^T\mathbf{A}\mathbf{x} > 0\) 对所有 \(\mathbf{x} \neq \mathbf{0}\)（正定）

正半定（PSD）则将条件 2 放宽为 \(\mathbf{x}^T\mathbf{A}\mathbf{x} \geq 0\)。

协方差矩阵 Σ 描述变量之间的关系：对角线是各变量的方差，非对角线是变量两两之间的协方差

PCA 的基础：对协方差矩阵做特征分解，找主成分方向

6.2 为什么 SPD 矩阵在统计学中如此重要？

SPD 矩阵几乎无处不在。协方差矩阵 \(\boldsymbol{\Sigma} = \mathbb{E}[(\mathbf{x} - \boldsymbol{\mu})(\mathbf{x} - \boldsymbol{\mu})^T]\) 出现在多元正态分布和主成分分析中。Gram 矩阵 \(\mathbf{X}^T\mathbf{X}\) 出现在最小二乘问题的正规方程 \(\mathbf{X}^T\mathbf{X}\boldsymbol{\beta} = \mathbf{X}^T\mathbf{y}\) 中，也出现在核方法（SVM、高斯过程）中。Hesse 矩阵（二阶导数矩阵）出现在优化的凸性判断和牛顿法中，还与统计中的 Fisher 信息矩阵密切相关。

SPD 矩阵的关键好处包括：保证 Cholesky 分解的存在性和唯一性、天然的数值稳定性，以及所有特征值为正（因此满秩）。

6.3 Cholesky 分解

定理：每个 SPD 矩阵 \(\mathbf{A}\) 都可以唯一分解为

\[ \mathbf{A} = \mathbf{LL}^T \]

其中 \(\mathbf{L}\) 是对角元素为正的下三角矩阵。

注意与 LU 分解的关系：对于 SPD 矩阵，可以选择 \(\mathbf{U} = \mathbf{L}^T\)，计算和存储只需 LU 分解的一半。

6.3.3 归纳证明的直觉

Cholesky 分解的存在性可以通过归纳法证明。核心思想是把矩阵分块：

\[ \mathbf{A} = \begin{pmatrix} a & \mathbf{c}^T \\ \mathbf{c} & \mathbf{B} \end{pmatrix} \]

6.6 带选主元的 Cholesky 分解

当矩阵是正半定（PSD）而非正定时，标准 Cholesky 可能失败（因为需要对零开方）。带选主元的 Cholesky 分解通过行列同时交换来处理这种情况：

\[ \mathbf{P}^T\mathbf{A}\mathbf{P} = \mathbf{LL}^T \]

其中 \(\mathbf{P}\) 是置换矩阵。这种方法能揭示矩阵的秩，并且对半定矩阵也稳定。

9 总结

9.1 知识地图

矩阵分解
├── 基本矩阵运算
│   ├── 加法: O(nm)
│   ├── 乘法: O(nmk)
│   └── 矩阵-向量乘法: O(n²)
│
├── 核心原则
│   └── 永远不要显式求逆矩阵！
│
├── 三角矩阵求解: O(n²)
│   ├── 前向替换 (下三角)
│   └── 回代 (上三角)
│
├── LU 分解: A = LU
│   ├── 思想：记录高斯消元步骤
│   ├── L：下三角（消元因子）
│   ├── U：上三角（消元结果）
│   ├── 复杂度：O(n³/3)
│   ├── 选主元：保证数值稳定
│   └── 多右端向量：分解一次，重复使用
│
├── Cholesky 分解: A = LL^T
│   ├── 仅适用于 SPD 矩阵
│   ├── 速度：约为 LU 的 2 倍
│   ├── 稳定性：天然稳定，无需选主元
│   └── 应用：
│       ├── SPD 系统求解
│       ├── 多元正态采样
│       └── 正规方程（最小二乘）
│
└── 方法选择
    ├── 一般矩阵 → LU（带选主元）
    ├── SPD 矩阵 → Cholesky
    └── 需要数值稳定性 → 考虑 QR 或 SVD（下章）

9.2 核心公式卡

概念 公式
LU 分解 \(\mathbf{A} = \mathbf{LU}\)，\(\mathbf{L}\) 下三角，\(\mathbf{U}\) 上三角
带选主元 \(\mathbf{PA} = \mathbf{LU}\)，\(\mathbf{P}\) 是置换矩阵
LU 求解 \(\mathbf{Ly} = \mathbf{b}\)（前向），\(\mathbf{Ux} = \mathbf{y}\)（回代）
Cholesky 分解 \(\mathbf{A} = \mathbf{LL}^T\)，\(\mathbf{A}\) 是 SPD
多元正态采样 \(\mathbf{y} = \boldsymbol{\mu} + \mathbf{Lz}\)，\(\mathbf{z} \sim \mathcal{N}(\mathbf{0}, \mathbf{I})\)
三角求解复杂度 \(O(n^2)\)
LU 分解复杂度 \(O(n^3/3)\)
Cholesky 分解复杂度 \(O(n^3/6)\)

9.3 实践指南

你遇到了什么 怎么做
一般线性方程组 \(\mathbf{Ax} = \mathbf{b}\) 用 solve(A, b)（内部使用带选主元的 LU）
SPD 矩阵的方程组 用 chol() + forwardsolve() + backsolve()
同一个 A，多个 b 先分解，再重复回代
需要 \(\mathbf{A}^{-1}\mathbf{b}\) 不要算 solve(A) %*% b，用 solve(A, b)
从 \(\mathcal{N}(\boldsymbol{\mu}, \boldsymbol{\Sigma})\) 采样 Cholesky 分解 \(\boldsymbol{\Sigma}\)，然后变换标准正态
chol() 报错 检查矩阵是否真的 SPD（特征值是否全正）
矩阵条件数很大 考虑正则化或使用更稳定的分解（QR、SVD）
