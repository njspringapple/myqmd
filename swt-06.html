Important敏捷宣言核心价值 (Kernwerte)

We are uncovering better ways of developing software by doing it and helping others do it. Through this work we have come to value:

更高价值 (links) 也有价值 (rechts)
个体和互动 (Individuals & Interactions) 流程和工具 (Processes & Tools)

响应变化 (Responding to Change) 遵循计划 (Following a Plan)

右侧的条目也有价值，但我们更重视左侧的条目。

2.3 极限编程的核心实践 | Praktiken aus Extreme Programming

极限编程 (XP) 推广了许多今天常用的敏捷方法。课程内容将这些实践归纳为五组。

第一组：频繁发布与增量规划——使用 User Stories 而非冗长的规格说明，频繁发布可运行版本，采用增量式计划。

第二组：客户深度参与——客户在团队中积极工作，负责定义验收测试 (Akzeptanztests)。

第三组：结对编程与集体所有权——实行结对编程 (Pair Programming)，代码归整个团队集体所有。

第四组：持续应对变化——定期向客户发布，采用测试驱动开发 (TDD)，减少技术债务 (Refactoring)，持续集成新功能。

第五组：保持简单——通过定期消除技术债务保持代码简洁，使用简单且聚焦的设计。

2.4 测试驱动开发 | Test-Driven Development (TDD)

Warning“Later is never” — 以后就是永远不会

程序员宁可写代码也不愿写测试。“以后再写”往往等于”永远不写”。测试工作枯燥，所以只会写最少的——导致不完整的测试。

测试应该是一等公民 (first-class citizens)！

# Draw cycle arrows
angles = [90, 210, 330]
labels_zh = ['写失败的测试\n(Red)', '写最少的代码\n使测试通过\n(Green)', '重构\n(Refactor)']
labels_de = ['Failing Test\nschreiben', 'Minimaler Code\nzum Bestehen', 'Code\naufräumen']
colors_tdd = ['#E74C3C', '#27AE60', '#3498DB']
emojis = ['', '', '']

radius = 3.2
for i, (angle, zh, de, color, emoji) in enumerate(zip(angles, labels_zh, labels_de, colors_tdd, emojis)):
    rad = np.radians(angle)
    x = radius * np.cos(rad) * 0.85
    y = radius * np.sin(rad) * 0.85

    circle = plt.Circle((x, y), 1.3, facecolor='white', edgecolor=color, linewidth=3)
    ax.add_patch(circle)
    ax.text(x, y + 0.5, emoji, ha='center', va='center', fontsize=18)
    ax.text(x, y - 0.15, zh, ha='center', va='center', fontsize=8, fontweight='bold', color=color,
            linespacing=1.3)

    # Arrow to next
    next_angle = angles[(i + 1) % 3]
    next_rad = np.radians(next_angle)
    nx = radius * np.cos(next_rad) * 0.85
    ny = radius * np.sin(next_rad) * 0.85

    mid_angle = (angle + next_angle) / 2
    if i == 0:
        mid_angle = (angle + next_angle + 360) / 2
    mid_rad = np.radians(mid_angle)

    # Simple arrow along the arc
    dx = nx - x
    dy = ny - y
    length = np.sqrt(dx**2 + dy**2)
    start_x = x + dx / length * 1.4
    start_y = y + dy / length * 1.4
    end_x = nx - dx / length * 1.4
    end_y = ny - dy / length * 1.4

    ax.annotate('', xy=(end_x, end_y), xytext=(start_x, start_y),
                arrowprops=dict(arrowstyle='->', color=color, lw=2.5,
                                connectionstyle='arc3,rad=0.3'))

ax.text(0, 0, 'TDD', ha='center', va='center', fontsize=16, fontweight='bold', color=COLORS['dark'])

结对编程中 Pilot 坐在键盘前，按 Copilot 的指令编写代码；Copilot 观察代码、提供指导和建议。两人大约每 20 分钟交换角色。其核心优势在于共同责任和知识在团队中的传播，四眼原则能有效减少错误，而两人共同决策也避免了低价值的重构。值得注意的是，生产力并不会降到 50%。

3.1.1 Scrum 团队 | Scrum Team

Scrum 团队由三个角色组成：Product Owner 负责 Product Backlog 和任务优先级排序；Developers 共同决定谁在什么时候做什么；Scrum Master 负责维护和支持 Scrum 流程的执行。团队在 Sprint Planning 中共同决定 Sprint 的范围，结果是 Sprint Backlog 和一个 Sprint Goal（与下一个增量相关的总目标）。Sprint 期间任务不再变更，且 Sprint 始终有固定的时间长度。

3.1.2 开发团队 | Development Team

开发团队不仅共同承担代码责任，还共同决定谁在什么时候做什么。整个规划都在团队中完成（实践中通常由个人做准备工作）。团队通过 Sprint Planning、Review 和 Retrospective 积极参与并塑造整个流程。

4.2 Daily Standup 三问 | Die drei Standup-Fragen

每次 Daily Standup 中，每位成员回答三个问题：

问题一：自上次 Standup 以来，我做了什么？ (Was habe ich seit dem letzten Standup gemacht?)

问题二：到下次 Standup 之前，我打算做什么？ (Was werde ich bis zum nächsten Standup machen?)

问题三：有什么障碍或风险？ (Welche Hindernisse oder Risiken gibt es?)

时间盒的核心优势在于：聚焦重点（Fokus auf das Wesentliche），减少收益递减的风险（diminishing Returns），以及定期反思和重新聚焦。

7.1 计划驱动与敏捷的结合 | Kombination plangesteuerter und agiler Methoden

对于需要长期资源规划（如三年跨度）的大型项目，可以将计划驱动方法与敏捷方法结合使用：先用类似瀑布模型的方式进行粗略的规格说明和系统设计，预留充足的变更风险空间，然后通过 Scrum 进行实际开发。当优先级变化或发现新需求时，分析其对长期计划的影响，必要时调整计划。
