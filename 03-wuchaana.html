2 误差度量：一切分析的起点

计算机不是精确的——它用有限的位数存储数字，每一步运算都可能引入微小的偏差。要分析这些偏差的影响，我们首先需要一种度量误差大小的方法。

2.1 标量的误差

对于标量，误差的定义很直观。设真实值为 \(v\)，近似值为 \(\tilde{v}\)：

绝对误差衡量”偏了多少”：

\[ \Delta_v = \tilde{v} - v \]

相对误差衡量”偏了百分之几”：

\[ \delta_v = \frac{|\tilde{v} - v|}{|v|} \]

相对误差的优点是无量纲，可以用百分比表示；缺点是真实值 \(v\) 不能为零。

两者之间有一个简洁的关系：

\[ \tilde{v} = v \cdot (1 + \delta_v') \]

其中 \(\delta_v' = (\tilde{v} - v)/v\) 是带符号的相对误差。

# 标量误差的例子
v_true <- pi
v_approx <- 3.14

# 注意：用近似值 减去 真实值，不要反了
abs_err <- v_approx - v_true
# 注意：相对误差需要用范数，对于标量类型，用绝对值
rel_err <- abs(abs_err) / abs(v_true)

2.2 向量和矩阵的误差

标量的概念自然推广到向量和矩阵——只需把绝对值替换成范数。

设 \(V\) 是带有范数 \(\|\cdot\|\) 的向量空间，\(\tilde{\mathbf{v}}\) 是 \(\mathbf{v}\) 的近似，则：

绝对误差（是一个向量/矩阵）：

\[ \Delta_{\mathbf{v}} = \tilde{\mathbf{v}} - \mathbf{v} \]

相对误差（是一个标量）：

\[ \delta_{\mathbf{v}} = \frac{\|\tilde{\mathbf{v}} - \mathbf{v}\|}{\|\mathbf{v}\|} = \frac{\|\Delta_{\mathbf{v}}\|}{\|\mathbf{v}\|} \]

若 \(\|\Delta_{\mathbf{v}}\| \leq \varepsilon\) 或 \(\delta_{\mathbf{v}} \leq \varepsilon\)，则称 \(\varepsilon\) 为绝对误差界或相对误差界。

2.3 误差界引理

相对误差不仅告诉我们”偏了多少”，还能给出近似值范数的上下界。这个引理在后续分析中非常有用。

引理：若 \(\delta_{\mathbf{z}} = \|\tilde{\mathbf{z}} - \mathbf{z}\| / \|\mathbf{z}\|\)，则

\[ \|\mathbf{z}\|(1 - \delta_{\mathbf{z}}) \leq \|\tilde{\mathbf{z}}\| \leq \|\mathbf{z}\|(1 + \delta_{\mathbf{z}}) \]

证明思路：由逆三角不等式 \(\big|\|\tilde{\mathbf{z}}\| - \|\mathbf{z}\|\big| \leq \|\tilde{\mathbf{z}} - \mathbf{z}\| = \delta_{\mathbf{z}}\|\mathbf{z}\|\)，展开绝对值即得。

# 用上面的例子验证引理
z <- v
z_tilde <- v_tilde
delta_z <- rel_err_v

norm_z <- sqrt(sum(z^2))
norm_z_tilde <- sqrt(sum(z_tilde^2))

lower <- norm_z * (1 - delta_z)
upper <- norm_z * (1 + delta_z)

cat("||z|| =", norm_z, "\n")
||z|| = 5 
cat("||z̃|| =", round(norm_z_tilde, 4), "\n")
||z̃|| = 5.36 
cat("下界: ||z||(1-δ) =", round(lower, 4), "\n")
下界: ||z||(1-δ) = 4.6394 
cat("上界: ||z||(1+δ) =", round(upper, 4), "\n")
上界: ||z||(1+δ) = 5.3606 
cat("\n验证:", round(lower, 4), "≤", round(norm_z_tilde, 4), "≤", round(upper, 4), "✓\n")

验证: 4.6394 ≤ 5.36 ≤ 5.3606 ✓

3 误差分解：误差从哪里来？

现在我们知道了如何度量误差，接下来的关键问题是：计算结果中的误差到底从哪里来？

3.1 计算的一般框架

设 \(f\) 是我们想要计算的数学函数（“问题”），\(\tilde{f}\) 是计算机上的近似算法。理想输入是 \(\mathbf{x}\)，但实际拿到的是有误差的 \(\tilde{\mathbf{x}}\)：

\[ \text{理想} \quad \mathbf{x} \xrightarrow{f} f(\mathbf{x}) \qquad \text{vs} \qquad \text{实际} \quad \tilde{\mathbf{x}} \xrightarrow{\tilde{f}} \tilde{f}(\tilde{\mathbf{x}}) \]

我们想要 \(f(\mathbf{x})\)，但得到的是 \(\tilde{f}(\tilde{\mathbf{x}})\)。差距有多大？

3.2 分解公式

总误差可以精确分解为两部分：

\[ \underbrace{\tilde{f}(\tilde{\mathbf{x}}) - f(\mathbf{x})}_{\text{总误差}} = \underbrace{\tilde{f}(\tilde{\mathbf{x}}) - f(\tilde{\mathbf{x}})}_{\text{算法误差}} + \underbrace{f(\tilde{\mathbf{x}}) - f(\mathbf{x})}_{\text{输入传播误差}} \]

中间巧妙地插入了 \(f(\tilde{\mathbf{x}})\)——用精确函数算带误差的输入，作为两部分的桥梁。

算法误差：即使输入没问题，算法本身（舍入、截断、近似）也会引入误差。这是算法的质量问题，我们可以通过设计更好的算法来控制。

输入传播误差：即使算法完美（用精确算术 \(f\)），输入本身的误差也会传播到输出。这是问题本身的性质，与算法无关。

# PPT 上的例子：计算 e^π
# 问题 f(x) = e^x，真实输入 x = π

4 条件数：问题本身有多敏感？

4.1 从直觉到定义

上一节告诉我们，即使算法完美，输入误差也会传播到输出。但不同的问题，传播的程度天差地别：有些问题”宽容”（输入偏一点，输出偏一点），有些问题”苛刻”（输入偏一点，输出偏十万八千里）。

条件数就是衡量这种”苛刻程度”的量。

4.2 形式化定义

设 \(f: \mathbb{R}^n \to \mathbb{R}^m\) 是一个问题，输入为 \(\mathbf{x}\)。

绝对条件数是满足以下不等式的最小常数 \(\kappa_{\text{abs}}\)：

\[ \|f(\tilde{\mathbf{x}}) - f(\mathbf{x})\| \leq \kappa_{\text{abs}} \|\tilde{\mathbf{x}} - \mathbf{x}\|, \quad \text{当 } \tilde{\mathbf{x}} \to \mathbf{x} \]

相对条件数是满足以下不等式的最小常数 \(\kappa_{\text{rel}}\)：

\[ \frac{\|f(\tilde{\mathbf{x}}) - f(\mathbf{x})\|}{\|f(\mathbf{x})\|} \leq \kappa_{\text{rel}} \frac{\|\tilde{\mathbf{x}} - \mathbf{x}\|}{\|\mathbf{x}\|}, \quad \text{当 } \tilde{\mathbf{x}} \to \mathbf{x} \]

换言之：输出的相对误差 ≤ 条件数 × 输入的相对误差。

条件数 含义 比喻
\(\kappa \leq 1\) 良态 输入误差不会被放大
\(\kappa \gg 1\) 病态 输入微小误差导致输出巨大偏差
\(\kappa = \infty\) 不适定 问题在该点无法可靠求解

4.3 标量函数的条件数

对于可微函数 \(f: \mathbb{R} \to \mathbb{R}\)，条件数有显式公式。由泰勒展开：

\[ f(\tilde{x}) - f(x) \approx f'(x)(\tilde{x} - x) \]

所以绝对条件数 \(\kappa_{\text{abs}} = |f'(x)|\)，相对条件数：

\[ \kappa_{\text{rel}} = |f'(x)| \cdot \frac{|x|}{|f(x)|} \]

# 标量函数条件数的例子

# 例1：f(x) = x^(-1)
# f'(x) = -x^(-2)，κ_abs = |f'(x)| = x^(-2)
# κ_rel = |f'(x)| · |x|/|f(x)| = x^(-2) · |x| / |x^(-1)| = 1
# 求倒数的相对条件数恒为 1——这是一个良态问题！

cat("f(x) = 1/x\n")

4.4 灾难性抵消：一个经典的病态问题

PPT 中给出了一个重要的例子：加法 \(f(\mathbf{x}) = x_1 + x_2\)。

写成矩阵形式：\(f(\mathbf{x}) = \mathbf{1}^\top \mathbf{x}\)，其中 \(\mathbf{1} = (1, 1)^\top\)。

绝对条件数：

\[ |f(\tilde{\mathbf{x}}) - f(\mathbf{x})| = |\mathbf{1}^\top(\tilde{\mathbf{x}} - \mathbf{x})| \leq \|\mathbf{1}\|_2 \|\tilde{\mathbf{x}} - \mathbf{x}\|_2 = \sqrt{2}\|\tilde{\mathbf{x}} - \mathbf{x}\|_2 \]

所以 \(\kappa_{\text{abs}} \leq \sqrt{2}\)——绝对条件数不大。

但相对条件数：

\[ \kappa_{\text{rel}} = \kappa_{\text{abs}} \frac{\|\mathbf{x}\|_2}{|f(\mathbf{x})|} \leq \sqrt{2} \frac{\|\mathbf{x}\|_2}{|x_1 + x_2|} \]

当 \(x_1 \approx -x_2\) 时，分母 \(|x_1 + x_2| \approx 0\)，\(\kappa_{\text{rel}} \to \infty\)！

这就是灾难性抵消（catastrophic cancellation）：两个大小相近但符号相反的数相加（实际上是相减），有效数字急剧丢失。

4.5 矩阵问题的条件数

PPT 中推导了求解 \(\mathbf{A}^{-1}\mathbf{x} = \mathbf{y}\) 时（即已知 \(\mathbf{x}\)，求 \(\mathbf{y} = \mathbf{A}^{-1}\mathbf{x}\)）的条件数。

设 \(f(\mathbf{x}) = \mathbf{A}^{-1}\mathbf{x}\)，则：

  • 利用范数的次可乘性性质 \[ \|f(\tilde{\mathbf{x}}) - f(\mathbf{x})\| = \|\mathbf{A}^{-1}(\tilde{\mathbf{x}} - \mathbf{x})\| \leq \|\mathbf{A}^{-1}\| \cdot \|\tilde{\mathbf{x}} - \mathbf{x}\| \]

所以，根据绝对条件数的定义， \(\kappa_{\text{abs}} = \|\mathbf{A}^{-1}\|\)。

对于相对条件数，经过推导（见上一章笔记的详细过程）：

\[ \kappa_{\text{rel}} \leq \|\mathbf{A}^{-1}\| \cdot \|\mathbf{A}\| = \kappa(\mathbf{A}) \]

这就是矩阵的条件数：

\[ \kappa(\mathbf{A}) = \|\mathbf{A}\| \cdot \|\mathbf{A}^{-1}\| \]

使用谱范数（2-范数）时：

\[ \kappa_2(\mathbf{A}) = \frac{\sigma_{\max}(\mathbf{A})}{\sigma_{\min}(\mathbf{A})} = \frac{\sqrt{\lambda_{\max}(\mathbf{A}^\top\mathbf{A})}}{\sqrt{\lambda_{\min}(\mathbf{A}^\top\mathbf{A})}} \]

5 稳定性：算法有多可靠？

5.1 从条件到稳定

上一节关注的是问题本身——“即使用精确算法，输入误差也会被放大”。

现在我们关注算法本身——“即使输入完美，算法也会引入误差”。

回忆误差分解：

\[ \underbrace{\tilde{f}(\tilde{\mathbf{x}}) - f(\mathbf{x})}_{\text{总误差}} = \underbrace{\tilde{f}(\tilde{\mathbf{x}}) - f(\tilde{\mathbf{x}})}_{\text{算法误差}} + \underbrace{f(\tilde{\mathbf{x}}) - f(\mathbf{x})}_{\text{输入传播误差}} \]

如果算法误差”小”，我们说算法是稳定的。

更精确地说，一个算法 \(\tilde{f}\) 是后向稳定的（backward stable），如果对任何输入 \(\mathbf{x}\)，算法的输出 \(\tilde{f}(\mathbf{x})\) 等于某个”稍微扰动的输入”的精确解：

\[ \tilde{f}(\mathbf{x}) = f(\mathbf{x} + \Delta\mathbf{x}), \quad \frac{\|\Delta\mathbf{x}\|}{\|\mathbf{x}\|} = O(\epsilon_{\text{machine}}) \]

也就是说：算法没有额外引入大误差，它只是精确求解了一个略有不同的问题。

5.2 样本方差：稳定 vs 不稳定

PPT 中给出了一个经典的例子。计算样本方差：

\[ f(\mathbf{x}) = \frac{1}{n}\sum_{i=1}^n (x_i - \bar{x})^2 \]

有两种等价的数学公式：

算法 1（稳定）：先减均值再平方

\[ \tilde{f}_1(\mathbf{x}) = \frac{1}{n}\sum_{i=1}^n (x_i - \bar{x})^2 \]

算法 2（不稳定）：用展开后的公式

5.2.1 方差公式的展开推导

从定义式出发：

\[ \text{Var} = \frac{1}{n}\sum_{i=1}^n (x_i - \bar{x})^2 \]

5.2.1.0.1 展开步骤

把平方项展开：

\[ (x_i - \bar{x})^2 = x_i^2 - 2x_i\bar{x} + \bar{x}^2 \]

代入求和：

\[ \frac{1}{n}\sum_{i=1}^n (x_i^2 - 2x_i\bar{x} + \bar{x}^2) \]

拆成三项：

\[ \frac{1}{n}\sum_{i=1}^n x_i^2 - \frac{2\bar{x}}{n}\sum_{i=1}^n x_i + \frac{1}{n}\sum_{i=1}^n \bar{x}^2 \]

5.2.1.0.2 化简每一项

第一项不变：(_{i=1}^n x_i^2)

第二项：因为 ({x} = {i=1}^n x_i)，所以 ({i=1}^n x_i = n{x})

\[ \frac{2\bar{x}}{n} \cdot n\bar{x} = 2\bar{x}^2 \]

第三项：({x}^2) 是常数，求和 n 次再除以 n

\[ \frac{1}{n} \cdot n\bar{x}^2 = \bar{x}^2 \]

5.2.1.0.3 最终结果

\[ \frac{1}{n}\sum_{i=1}^n x_i^2 - 2\bar{x}^2 + \bar{x}^2 = \frac{1}{n}\sum_{i=1}^n x_i^2 - \bar{x}^2 \]

这就是第二个公式。

\[ \tilde{f}_2(\mathbf{x}) = \frac{1}{n}\sum_{i=1}^n x_i^2 - \bar{x}^2 \]

数学上完全等价，但数值上天差地别！

5.3 与机器学习的联系：SGD 的稳定性

PPT 提到了随机梯度下降（SGD）作为稳定性概念的实际例子。训练神经网络时：

概念 对应 SGD
问题 \(f\) 求损失函数 \(L(\theta)\) 的最小值
算法 \(\tilde{f}\) 有限精度的 SGD 迭代
算法误差 舍入误差 + mini-batch 估计误差的累积

学习率太大时，梯度估计中的误差会在每次迭代中被放大——这就是算法不稳定。

6 复合算法的稳定性

6.1 核心定理

大多数算法不是一步完成的，而是分步进行：\(f = h \circ g\)。PPT 给出了一个关键定理，告诉我们误差在子步骤间如何传播。

定理：设 \(f(\mathbf{x}) = h(g(\mathbf{x}))\)，算法 \(\tilde{f} = \tilde{h} \circ \tilde{g}\)，则

\[ \frac{\|\tilde{f}(\mathbf{x}) - f(\mathbf{x})\|}{\|f(\mathbf{x})\|} \leq \frac{\|\tilde{h}(\tilde{\mathbf{y}}) - h(\tilde{\mathbf{y}})\|}{\|h(\mathbf{y})\|} + \kappa_{\text{rel}}(h, g(\mathbf{x})) \cdot \frac{\|\tilde{g}(\mathbf{x}) - g(\mathbf{x})\|}{\|g(\mathbf{x})\|} \]

其中 \(\mathbf{y} = g(\mathbf{x})\)，\(\tilde{\mathbf{y}} = \tilde{g}(\mathbf{x})\)。

直观理解：总的相对误差由两部分组成。第一项是 \(\tilde{h}\) 本身的算法误差；第二项是 \(\tilde{g}\) 的误差经过 \(h\) 放大后的结果，放大倍数就是 \(h\) 在 \(g(\mathbf{x})\) 处的条件数。

关键推论：如果 \(h\) 的条件数很大，它会放大前面 \(\tilde{g}\) 的所有误差。因此，条件数差的步骤应该尽早执行——放在最前面时，后面没有高条件数步骤来放大它的误差。

# 稳定的二次方程求解
stable_quadratic <- function(a, b, c_val) {
  disc <- sqrt(b^2 - 4 * a * c_val)
  # 选择不发生抵消的根
  if (b >= 0) {
    x1 <- (-b - disc) / (2 * a)
  } else {
    x1 <- (-b + disc) / (2 * a)
  }
  x2 <- c_val / (a * x1)  # 韦达定理
  c(x1, x2)
}

9 总结

9.1 知识地图

数值误差分析
├── 误差度量
│   ├── 绝对误差 Δ = ṽ - v
│   ├── 相对误差 δ = ||Δ|| / ||v||
│   └── 误差界引理：||v||(1-δ) ≤ ||ṽ|| ≤ ||v||(1+δ)
│
├── 误差分解
│   ├── 总误差 = 算法误差 + 输入传播误差
│   ├── 算法误差 → 衡量算法质量 → 稳定性
│   └── 传播误差 → 衡量问题难度 → 条件数
│
├── 条件数（问题的性质）
│   ├── 绝对条件数 κ_abs
│   ├── 相对条件数 κ_rel
│   ├── 矩阵条件数 κ(A) = ||A|| · ||A⁻¹|| = σ_max/σ_min
│   └── 灾难性抵消：大数相减的 κ_rel → ∞
│
├── 稳定性（算法的性质）
│   ├── 后向稳定：算法误差 ≈ ε_machine
│   ├── 复合定理：κ(h) 放大 g 的误差
│   └── 原则：条件数差的步骤尽早执行
  • 条件（Kondition）描述问题对输入误差的敏感性
  • 稳定性（Stabilität）描述算法对误差的鲁棒性
  • 条件数差的步骤应尽早执行！
