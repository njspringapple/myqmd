1.2 建立模型：一个简单的猜测

Modellaufbau: Eine einfache Vermutung

2.3 如何度量”整体误差”——范数的直觉

Wie misst man den „Gesamtfehler”? — Intuition der Norm

每个样本的 残差（预测值 - 真实值）为：

\[ \epsilon_i = w \cdot \text{Area}_i - \text{Price}_i \]

所有残差组成一个向量 \(\boldsymbol{\epsilon} = (\epsilon_1, \epsilon_2, \ldots, \epsilon_N)^\top\)。我们需要一个 单一数字 来描述”这个向量整体有多大”。

这个数字就叫 范数（Norm）——可以理解为向量”长度”的推广。

3.1 损失函数：机器学习的”评分标准”

Verlustfunktion: Das „Bewertungssystem” des maschinellen Lernens

给定权重 \(w\)，我们可以计算一个分数来衡量模型的好坏。这个分数就叫 损失函数（Verlustfunktion / Loss Function）。

3.1.1 SSE——平方误差之和

\[ L_{\text{SSE}}(w) = \sum_{i=1}^{N} (w \cdot \text{Area}_i - \text{Price}_i)^2 \]

3.1.2 MSE——平均平方误差（更常用）

\[ L_{\text{MSE}}(w) = \frac{1}{N} \sum_{i=1}^{N} (w \cdot \text{Area}_i - \text{Price}_i)^2 \]

两者关系：\(L_{\text{MSE}} = \frac{1}{N} L_{\text{SSE}}\)。由于 \(\frac{1}{N}\) 是常数，两者最小值点相同——但 MSE 不受样本量影响，更适合比较不同数据集。

结合以上范数和损失函数定义，不难看出：

L1_norm <- function(r) sum(abs(r))
L2_norm <- function(r) sqrt(sum(r^2))
L2_sq <- function(r) sum(r^2)

# 定义损失函数 | Verlustfunktionen definieren
loss_sae <- function(w) L1_norm(w * Area - Price) # 绝对误差和
loss_sse <- function(w) L2_sq(w * Area - Price)   # 平方误差和
loss_mse <- function(w) L2_sq(w * Area - Price) / length(Area)  # 均方误差
loss_rmse <- function(w) L2_norm(w * Area - Price) / sqrt(length(Area)) # 均方根误差

3.1.3 实践中的选择 | Auswahl in der Praxis

场景 / Szenario 推荐 / Empfohlen 原因 / Grund
理论推导 SSE 公式更简洁
模型比较 MSE 与样本量无关，可比较不同数据集
报告结果 \(\text{RMSE} = \sqrt{\text{MSE}}\) 与原始数据单位相同，易解释

4.3 几何直觉：投影视角

问题：找 \(\mathbf{w}\) 使得 \(X\mathbf{w}\) 尽可能接近 \(\mathbf{y}\)。

当 \(\mathbf{w}\) 遍历所有可能取值时，\(X\mathbf{w}\) 可以写成各个特征列的线性组合：\(X\mathbf{w}=w_1\mathbf{x}{(1)}+\cdots+w_p\mathbf{x}{(p)}\)（其中 \(\mathbf{x}{(j)}\) 是 \(X\) 的第 \(j\) 列）。因此，所有可能的预测向量 \(X\mathbf{w}\) 都落在由这些特征列张成的空间里——这就是矩阵 \(X\) 的 列空间 \(\text{Col}(X)=\text{span}{\mathbf{x}{(1)},\ldots,\mathbf{x}_{(p)}}\)。

几何答案：\(\mathbf{y}\) 在这个“由特征列张成的空间” \(\text{Col}(X)\) 上的 正交投影，就是所有可达预测 \(X\mathbf{w}\) 中与 \(\mathbf{y}\) 距离最近的那个点（也就是最优预测 \(\hat{\mathbf{y}}\)）。

正交条件 \(X^T(\mathbf{y} - X\hat{\mathbf{w}}) = \mathbf{0}\) 整理后得到 正规方程：

\[ X^T X \hat{\mathbf{w}} = X^T \mathbf{y} \]

当 \(X^T X\) 可逆时，一步到位：

\[ \boxed{\hat{\mathbf{w}} = (X^T X)^{-1} X^T \mathbf{y}} \]

Tip🔍 为什么正交条件里有 \(X^T\)？（点击展开）

列空间 \(\text{Col}(X)\) 由 \(X\) 的所有列张成。残差 \(\mathbf{r}\) 必须与 每一列 都垂直：

\[ \mathbf{x}_1^T \mathbf{r} = 0,\quad \mathbf{x}_2^T \mathbf{r} = 0,\quad \ldots,\quad \mathbf{x}_p^T \mathbf{r} = 0 \]

把这 \(p\) 个内积条件紧凑地写在一起，就是 \(X^T \mathbf{r} = \mathbf{0}\)。

这里 \(X^T\) 的作用是把 \(X\) 的每一列”竖起来”变成行，然后分别与 \(\mathbf{r}\) 做点积，检验是否都为零。

\(X^T\) wandelt jede Spalte von \(X\) in eine Zeile um und bildet dann jeweils das Skalarprodukt mit \(\mathbf{r}\), um zu prüfen, ob alle null sind.

4.4 解析解的局限：为什么不能一招鲜吃遍天？

Grenzen der analytischen Lösung

Warning⚠️ 三个致命限制 | Drei fatale Einschränkungen
  1. 计算代价高：\((X^TX)^{-1}\) 的计算量为 \(O(p^3)\)，当特征数 \(p\) 达到百万级时根本不可行
  2. 数值不稳定：当 \(X^TX\) 接近奇异（某些特征高度相关），求逆会产生巨大误差
  3. 只适用于线性模型：神经网络等非线性模型没有闭式解

结论：我们需要一种 通用的迭代方法——梯度下降。

Fazit: Wir brauchen eine allgemeine iterative Methode — den Gradientenabstieg.

5.1 核心直觉：蒙眼下山

Kernintuition: Blindes Bergabgehen

想象你被蒙上眼睛，放在一个山谷里，只能用脚感知地面的坡度。你的策略是什么？

“感受坡度最陡的方向，朝那个方向迈一步。重复。”

这就是梯度下降！

  • 梯度 \(\nabla L(\mathbf{w})\) = 告诉你”地面往哪个方向最陡地上升”
  • 负梯度 \(-\nabla L(\mathbf{w})\) = “下降最快的方向”
  • 学习率 \(\eta\) = 每一步迈多大

更新公式：

\[ \boxed{\mathbf{w}^{(t+1)} = \mathbf{w}^{(t)} - \eta \,\nabla L(\mathbf{w}^{(t)})} \]

5.2 梯度的计算

Berechnung des Gradienten

对于 MSE 损失函数 \(L(\mathbf{w}) = \frac{1}{N}\|X\mathbf{w} - \mathbf{y}\|_2^2\)，梯度为：

\[ \nabla L(\mathbf{w}) = \frac{2}{N} X^T(X\mathbf{w} - \mathbf{y}) \]

对于 SSE 损失函数 \(L(\mathbf{w}) = \|X\mathbf{w} - \mathbf{y}\|_2^2\)，梯度为：

\[ \nabla L(\mathbf{w}) = 2 X^T(X\mathbf{w} - \mathbf{y}) \]

# 核心函数 | Kernfunktionen

# 预测函数 | Vorhersagefunktion
预测 <- function(X, w) {
  return(X %*% w)
}

# MSE 损失函数 | MSE-Verlustfunktion
# 损失函数在这里的作用是：衡量当前 w 好不好
损失函数 <- function(X, y, w) {
  残差 <- 预测(X, w) - y
  return(mean(残差^2))
}

# 因为损失函数 L(w) 虽然是一个标量，但它是由 p 个权重共同决定的：
# w = (w1, …, wp)。你想让损失变小，就必须回答“每一个权重该往上调还是往下调、
# 调多少才最能让损失下降”。
# 用得到的下降的梯度向量来更新初始权重向量 w
# MSE 梯度（解析公式）| MSE-Gradient (analytische Formel)
梯度_解析 <- function(X, y, w) {
  N <- nrow(X)
  残差 <- 预测(X, w) - y
  梯度 <- (2 / N) * t(X) %*% 残差
  return(as.vector(梯度))
}

学习率（learning rate）\(\eta\) 是梯度下降里的一个超参数：它控制每一步沿着“负梯度方向”更新权重的步长。

对 MSE 损失函数 \[ L(\mathbf{w})=\frac{1}{N}\lVert X\mathbf{w}-\mathbf{y}\rVert_2^2, \] 其梯度为 \[ \nabla_{\mathbf{w}}L(\mathbf{w})=\frac{2}{N}X^\top(X\mathbf{w}-\mathbf{y}). \]

梯度下降的更新公式是： \[ \mathbf{w}^{(t+1)}=\mathbf{w}^{(t)}-\eta\,\nabla_{\mathbf{w}}L\!\left(\mathbf{w}^{(t)}\right). \]

也就是说在每一次迭代 \(t\)： - 先计算当前梯度 \(\nabla_{\mathbf{w}}L(\mathbf{w}^{(t)})\) - 再用学习率 \(\eta\) 决定更新步长，沿负梯度方向移动

6.1 为什么需要验证？

Warum ist eine Überprüfung notwendig?

在复杂模型中，手推梯度公式容易出错。我们需要一种方法来验证梯度计算是否正确。

Bei komplexen Modellen ist es leicht, Fehler bei der manuellen Herleitung der Gradientenformel zu machen.

6.2 方案 1：朴素方法——数值微分

Ansatz 1: Naive Methode — Numerische Differentiation

直觉：就像调收音机——对每个旋钮，往右拧一点看效果，往左拧一点看效果，从而估计这个旋钮对信号的影响。

\[ \frac{\partial L}{\partial w_j} \approx \frac{L(\mathbf{w} + \epsilon \, \mathbf{e}_j) - L(\mathbf{w} - \epsilon \, \mathbf{e}_j)}{2\epsilon} \]

其中 \(\mathbf{e}_j\) 是第 \(j\) 个方向的单位向量，\(\epsilon\) 是微小扰动（如 \(10^{-5}\)）。

# --- 朴素方法（数值微分）---
# 对每个旋钮，拧一点点，看损失变化多少
# 需要算 2p 次损失函数

朴素方法计算梯度 <- function(X, y, w, epsilon = 1e-5) {
  梯度 <- numeric(length(w))
  计算次数 <- 0
  
  for (j in 1:length(w)) {
    # 把第 j 个旋钮往右拧一点
    w_plus <- w
    w_plus[j] <- w_plus[j] + epsilon
    L_plus <- 损失函数(X, y, w_plus)
    计算次数 <- 计算次数 + 1
    
    # 把第 j 个旋钮往左拧一点
    w_minus <- w
    w_minus[j] <- w_minus[j] - epsilon
    L_minus <- 损失函数(X, y, w_minus)
    计算次数 <- 计算次数 + 1
    
    # 数值微分：(L_plus - L_minus) / (2 * epsilon)
    梯度[j] <- (L_plus - L_minus) / (2 * epsilon)
  }
  
  attr(梯度, "计算次数") <- 计算次数
  return(梯度)
}

问题：需要计算 \(2p\) 次损失函数（\(p\) 是参数数量）。当模型有百万参数时，这太慢了！

Problem: \(2p\) Verlustfunktionsauswertungen nötig — bei Millionen Parametern viel zu langsam!

6.3 方案 2：解析梯度——数学公式直接算

Ansatz 2: Analytischer Gradient — Direkte mathematische Formel

直觉：利用数学推导，直接写出梯度的闭式表达式。

对于 MSE 损失函数，梯度有闭式解：

\[ \nabla L(\mathbf{w}) = \frac{2}{N} X^T (X\mathbf{w} - \mathbf{y}) \]

一次矩阵运算就搞定，不管有多少参数！

# --- 解析梯度（直接用数学公式）---
# 对于均方误差损失，梯度有闭式解：
# grad_L = (2/N) * X^T * (X*w - y)
# 一次矩阵运算就搞定

解析方法计算梯度 <- function(X, y, w) {
  预测值 <- 预测(X, w)
  误差 <- 预测值 - y
  梯度 <- (2 / nrow(X)) * t(X) %*% 误差
  return(as.vector(梯度))
}

6.7 实践建议：何时用哪种方法

朴素方法用于：调试和验证 — 当你推导了一个新的梯度公式时，用朴素方法（数值微分）检验它是否正确。就像考试时用另一种方法验算答案。

解析方法用于：实际训练 — 一旦确认公式正确，训练时永远使用解析梯度。

Analytische Methode für: Tatsächliches Training — Sobald die Formel als korrekt bestätigt ist, verwenden Sie beim Training immer den analytischen Gradienten.

验证流程 | Verifizierungsworkflow:
  1. 推导解析梯度公式
  2. 实现解析梯度函数
  3. 用朴素方法（数值微分）计算同一点的梯度
  4. 比较两者差异（应 < 1e-5）
  5. 确认无误后，正式训练只用解析方法

7.2 核心洞察：梯度 = 期望

Schlüsselerkenntnis: Gradient = Erwartungswert

全批量梯度实际上是所有样本梯度的 平均（即期望）：

\[ \nabla L(\mathbf{w}) = \frac{1}{N}\sum_{i=1}^{N} \nabla\ell_i(\mathbf{w}) = \mathbb{E}_{i \sim \text{Uniform}}\!\big[\nabla\ell_i(\mathbf{w})\big] \]

其中 \(\ell_i(\mathbf{w}) = (\mathbf{x}_i^T \mathbf{w} - y_i)^2\) 是第 \(i\) 个样本的损失。

既然是平均，我们就可以用随机采样来估计它！ 就像做民意调查不需要问全国所有人一样。

Da es ein Durchschnitt ist, können wir ihn durch Zufallsstichproben schätzen! Wie bei einer Meinungsumfrage.

7.3 三种梯度下降策略

Drei Gradientenabstiegsstrategien

7.3.1 SGD：每次只用 1 个样本

\[ \mathbf{w}^{(t+1)} = \mathbf{w}^{(t)} - \eta \,\nabla \ell_{i_t}(\mathbf{w}^{(t)}) \]

7.3.2 Mini-batch GD：每次用 \(B\) 个样本（折中方案）

\[ \mathbf{w}^{(t+1)} = \mathbf{w}^{(t)} - \eta \cdot \frac{1}{B} \sum_{i \in \mathcal{B}_t} \nabla \ell_i(\mathbf{w}^{(t)}) \]

# 通用 Mini-batch GD（batch_size = 1 即 SGD，= N 即 Full-batch）
# Allgemeiner Mini-Batch-GD (batch_size = 1 → SGD, = N → Full-Batch)
minibatch_gd <- function(X, y, w_init, lr, batch_size, max_iter) {
  w <- w_init; N <- nrow(X)
  loss_hist <- numeric(max_iter)
  for (i in 1:max_iter) {
    idx <- sample(N, batch_size)
    Xb <- X[idx, , drop = FALSE]; yb <- y[idx]
    grad <- (2 / batch_size) * as.vector(t(Xb) %*% (Xb %*% w - yb))
    w <- w - lr * grad
    loss_hist[i] <- 损失函数(X, y, w)
  }
  list(w = w, loss = loss_hist)
}

8.3 标准化公式

Standardisierungsformel

将每个特征转换为均值为 0、标准差为 1 的分布：

\[ x_j^{\text{std}} = \frac{x_j - \mu_j}{\sigma_j} \]

其中 \(\mu_j = \frac{1}{N}\sum_{i=1}^N x_{ij}\) 是均值，\(\sigma_j = \sqrt{\frac{1}{N}\sum_{i=1}^N (x_{ij} - \mu_j)^2}\) 是标准差。

# 标准化函数 | Standardisierungsfunktion
standardize <- function(X) {
  mu <- colMeans(X)
  sigma <- apply(X, 2, sd)
  X_std <- sweep(sweep(X, 2, mu), 2, sigma, "/")
  list(X_std = X_std, mu = mu, sigma = sigma)
}

# Vollständiges Training mit Konvergenzprüfung und Fortschrittsbericht
gd_full <- function(X, y, w_init, lr, max_iter, tol = 1e-8) {
  w <- w_init
  loss_h <- numeric(max_iter)
  
  for (i in 1:max_iter) {
    grad <- 梯度_解析(X, y, w)
    w_new <- w - lr * grad
    loss_h[i] <- 损失函数(X, y, w_new)
    
    # 检查收敛 | Konvergenz prüfen
    if (sqrt(sum((w_new - w)^2)) < tol) {
      cat("  在第", i, "次迭代收敛 | Konvergiert bei Iteration", i, "\n")
      loss_h <- loss_h[1:i]; w <- w_new; break
    }
    
    # 进度报告 | Fortschrittsbericht
    if (i %% 500 == 0) {
      cat("  迭代", i, "- MSE:", format(loss_h[i], scientific = TRUE), "\n")
    }
    w <- w_new
  }
  list(w = w, loss = loss_h, iters = length(loss_h))
}

10.1 核心公式完全解读

现在让我们重新审视开篇那个”吓人”的公式——每个符号你都认识了：

\[ \hat{\mathbf{w}} = \underbrace{\arg\min_{\mathbf{w} \in \mathbb{R}^p}}_{\text{找最优参数}} \underbrace{\frac{1}{N} \sum_{i=1}^{N}}_{\text{N个样本取平均}} \underbrace{\bigl(f(\mathbf{x}_i;\, \mathbf{w}) - y_i\bigr)^2}_{\text{每个样本的预测误差的平方}} \]

符号 / Symbol 本章含义 / Bedeutung 第一次遇到 / Wo eingeführt
\(\mathbf{w} \in \mathbb{R}^p\) 权重向量 (3维) §1 完美世界
\(N\) 10000 个样本 §2 引入噪声
\(f(\cdot;\mathbf{w})\) \(X\mathbf{w}\) 线性模型 §4 多特征
\((\cdot - y_i)^2\) 平方损失 §3 损失函数
\(\frac{1}{N}\sum\) MSE 取平均 §3 SSE vs MSE
\(\arg\min\) 梯度下降 / 正规方程 §4 & §5
