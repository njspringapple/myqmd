2.1.1 坏例子 | Schlechtes Beispiel

不同抽象层次混在同一个方法中——既有高层的 databaseContainsStudent 调用，又有低层的正则表达式细节：

public void registerStudentBad(String name, String matriculationNumber) {
    if (name == null || name.trim().isEmpty()) {          // ⚡ LOW-LEVEL
        throw new IllegalArgumentException("Name cannot be empty");
    }
    Pattern pattern = Pattern.compile("^\\d{7}$");        // ⚡ LOW-LEVEL
    if (!pattern.matcher(matriculationNumber).matches()) {
        throw new IllegalArgumentException("...");
    }
    if (databaseContainsStudent(name)) {                  // higher level
        throw new IllegalStateException("...");
    }
    persistStudent(name, matriculationNumber);             // higher level
    sendWelcomeEmail(name);                                // higher level
}

2.1.2 好例子 | Gutes Beispiel

每一层只做一个抽象层次的事情，读起来像”为了……，我们……“的段落 (UM-ZU-Absätze)：

public void validateAndRegisterStudent(String name, String matriculationNumber) {
    validateStudentData(name, matriculationNumber);
    checkForDuplicates(name);
    persistStudent(name, matriculationNumber);
    sendWelcomeEmail(name);
}

private void validateStudentData(String name, String matriculationNumber) {
    validateName(name);
    validateMatriculationNumber(matriculationNumber);
}

private void validateName(String name) {
    if (name == null || name.trim().isEmpty()) {
        throw new IllegalArgumentException("Name cannot be empty");
    }
}

private void validateMatriculationNumber(String matriculationNumber) {
    Pattern pattern = Pattern.compile("^\\d{7}$");
    if (!pattern.matcher(matriculationNumber).matches()) {
        throw new IllegalArgumentException("...");
    }
}
ImportantUM-ZU-Absätze (为了…段落)

好代码可以用”为了……”的句式来描述：

  • 为了验证并注册学生，我们验证学生数据、检查重复、保存学生、发送欢迎邮件。
  • 为了验证学生数据，我们验证名字和学号。
  • 为了验证名字，我们检查名字不为空且至少 2 个字符。

每个段落描述一个抽象层次。

2.4 命令与查询分离 | Anweisung und Abfrage trennen

函数要么执行操作 (Command)，要么返回信息 (Query)，不应两者兼做。

// 坏：remove 既是 Command 又是 Query
if (students.remove(student)) {
    System.out.println("Student removed");
}

// 好：分离 Query 和 Command
if (students.contains(student)) {          // Query
    students.remove(student);               // Command
    System.out.println("Student removed");
}

3.2 迪米特法则 | Law of Demeter

Warning“不要和陌生人说话” — “Don’t talk to strangers”

一个对象只应调用以下对象的方法：自身 (this)、参数对象、自己创建的对象、直接成员变量。

避免：object.getX().getY().getZ() 这种链式调用。

6.3 Checked vs. Unchecked Exceptions

6.3.1 Checked Exceptions (Java 特有)

声明在方法签名上，调用者必须处理或继续向上抛出：

// Java 标准库：BufferedReader
public class BufferedReader {
    public String readLine() throws IOException { /* .. */ }
}

// 调用者选择 1：处理
private void handlingMethod() {
    try (BufferedReader reader = new BufferedReader(new FileReader("students.txt"))) {
        String line = reader.readLine();
    } catch (IOException e) {
        // .. handle e ..
    }
}

// 调用者选择 2：继续向上抛出
private void propagatingMethod() throws IOException {
    try (BufferedReader reader = new BufferedReader(new FileReader("students.txt"))) {
        String line = reader.readLine();
    }
}
WarningChecked Exception 的传播问题

如果最底层 loadConfigurationFile() 抛出 IOException，那么整个调用链上每一层都必须声明 throws IOException——这违反了抽象 (Abstraktionsverletzung)。如果将来不再抛出 IOException，所有调用者都得修改。

6.3.2 Unchecked Exceptions

不需要声明，不需要强制处理（但不处理就会程序崩溃）。通常通过 Javadoc 中的 @throws 来文档化。

/**
 * @throws NullPointerException if student is null
 * @throws IllegalStateException if the student has no finished courses
 */
public double calculateAverageGrade(Student student) {
    if (student == null) {
        throw new NullPointerException("Student cannot be null");
    }
    // ...
}
