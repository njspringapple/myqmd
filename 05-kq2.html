最小二乘法：从直觉到算法（数值线代·一课版）

1 课前 10 分钟速通：本章“够用就行”的前置知识

本章目标：不靠背诵公式，通过动手 + 观察建立最小二乘的几何直觉与算法选择能力。
你只需要掌握下面 4 件事（每件都有 1 个可运行的检验）。

1.1 1. 向量 2-范数与“距离”

我们用 \(\|\cdot\|_2\) 表示欧氏长度。最小二乘就是在找“离 \(\mathbf{b}\) 最近的 \(\mathbf{Ax}\)”。

1.2 2. 正交矩阵：不改变长度（数值稳定的关键）

若 \(\mathbf{Q}^T\mathbf{Q}=\mathbf{I}\)，则对任意 \(\mathbf{z}\)：\(\|\mathbf{Qz}\|_2=\|\mathbf{z}\|_2\)。
这意味着“乘一个正交矩阵”不会放大误差，也不会改变最小化问题的解。

1.3 3. 条件数：一句话就够

\(\kappa_2(\mathbf{A})=\sigma_{\max}/\sigma_{\min}\) 大 \(\Rightarrow\) 问题对扰动更敏感（更容易“翻车”）。
数值线代里：我们关心的不只是“数学上对”，更是“浮点数下稳不稳”。

1.4 4. 三个分解只记用途（不记定义也能用）

  • 正规方程 + Cholesky：快，但会形成 \(\mathbf{A}^T\mathbf{A}\)，可能把条件数平方化
  • QR（Householder）：默认首选，稳定
  • SVD：最稳，能处理（近）秩亏，但更慢
# 基础工具函数（尽量少、尽量通用）

vnorm <- function(x) sqrt(sum(drop(x)^2))

kappa2 <- function(A, tol = 0) {
  sv <- svd(A, nu = 0, nv = 0)
  d <- sv$d
  if (length(d) == 0) return(NA_real_)
  if (tol > 0) d <- d[d > tol * max(d)]
  if (length(d) == 0) return(Inf)
  max(d) / min(d)
}

ls_residual <- function(A, x, b) b - A %*% x

relerr <- function(x_hat, x_true) vnorm(x_hat - x_true) / max(vnorm(x_true), .Machine$double.eps)

2 学习导航：这一章你要会做的 3 件事

  1. 直觉：最小二乘 = 把 \(\mathbf{b}\) 正交投影到 \(\text{col}(\mathbf{A})\)
  2. 判据：残差 \(\mathbf{r}=\mathbf{b}-\mathbf{A}\hat{\mathbf{x}}\) 满足 \(\mathbf{A}^T\mathbf{r}=0\)
  3. 算法选择（一课版结论先给）：
    • 默认用 QR（qr.solve / lm）
    • \(\mathbf{A}\)（近）秩亏或很病态：用 SVD（或正则化）
    • \(\kappa\) 很小且要对很多不同 \(\mathbf{b}\) 重复求解：可以考虑 Cholesky/正规方程（快，但要小心）

4 核心直觉：最小二乘 = 投影

4.1 一句话

\(\mathbf{A}\hat{\mathbf{x}}\) 是 \(\mathbf{b}\) 在 \(\text{col}(\mathbf{A})\) 上的正交投影，
残差 \(\mathbf{r}=\mathbf{b}-\mathbf{A}\hat{\mathbf{x}}\) 垂直于列空间。

5 从直觉到可计算：正规方程 \(A^T r=0\)

5.1 推导（只保留“可用的结论”）

残差 \(\mathbf{r}=\mathbf{b}-\mathbf{A}\hat{\mathbf{x}}\) 垂直于列空间
\(\Rightarrow\) 对 \(\mathbf{A}\) 的每一列 \(\mathbf{a}_j\)：\(\mathbf{a}_j^T\mathbf{r}=0\)
\(\Rightarrow\) \[ \mathbf{A}^T(\mathbf{b}-\mathbf{A}\hat{\mathbf{x}})=0 \quad\Longleftrightarrow\quad \mathbf{A}^T\mathbf{A}\hat{\mathbf{x}}=\mathbf{A}^T\mathbf{b}. \]

数值线代观点：正规方程是“数学上正确的条件”，但不一定是“数值上好的算法”。

6 重要警告：正规方程会“平方化”条件数（为什么 QR 更稳）

这一节只想让你形成可操作的直觉：
“形成 \(A^TA\) 会让问题更敏感”，所以默认不要用它当首选算法。

6.1 关键事实（不用背推导）

\[ \kappa_2(\mathbf{A}^T\mathbf{A}) \approx \kappa_2(\mathbf{A})^2. \]

这意味着：如果 \(\kappa_2(A)=10^8\)，那么 \(\kappa_2(A^TA)\approx 10^{16}\)（双精度的极限），算法可能直接失去有效数字。

6.2 动手 3（8 分钟）：一个“短反例”——NE vs QR 在病态矩阵上

我们构造一个有精确解的最小二乘问题（\(\mathbf{b}\in\text{col}(A)\)），这样比较更干净：差异主要来自数值稳定性，而不是拟合误差。

展开代码 
set.seed(2)
m <- 40; n <- 6

ls_svd <- function(A, b, tol = 1e-12) {
  sv <- svd(A)
  d <- sv$d
  keep <- d > tol * max(d)
  Utb <- drop(t(sv$u) %*% b)
  coeff <- rep(0, length(d))
  coeff[keep] <- Utb[keep] / d[keep]
  x <- sv$v %*% coeff
  drop(x)
}

# 用 SVD 构造指定条件数的 A
make_A_with_kappa <- function(m, n, kappa_target) {
  U <- qr.Q(qr(matrix(rnorm(m*m), m)))
  V <- qr.Q(qr(matrix(rnorm(n*n), n)))
  sig <- exp(seq(log(1), log(kappa_target), length.out = n))
  U[,1:n] %*% diag(sig) %*% t(V)
}

k_target <- 1e12
A_bad <- make_A_with_kappa(m, n, k_target)
x_true <- rep(1, n)
b_exact <- A_bad %*% x_true

# 1) 正规方程解
x_ne_bad <- tryCatch(
  solve(crossprod(A_bad), crossprod(A_bad, b_exact)),
  error = function(e) rep(NA_real_, n)
)

# 2) QR 解（推荐）
x_qr_bad <- tryCatch(
  qr.solve(A_bad, b_exact),
  error = function(e) {
    message("QR 失败（数值上秩亏/极病态），回退到 SVD。")
    ls_svd(A_bad, b_exact, tol = 1e-12)
  }
)

7 三种算法：你需要会用、会选（不要求手搓实现）

7.1 方法 A：正规方程 + Cholesky（快，但有风险）

  • 适合：\(\kappa(A)\) 小、对很多不同 \(\mathbf{b}\) 反复求解、追求速度
  • 风险：条件数平方化；病态时不可靠
展开代码 
ls_chol <- function(A, b) {
  AtA <- crossprod(A)
  Atb <- crossprod(A, b)
  R <- chol(AtA)              # AtA = R^T R
  y <- forwardsolve(t(R), Atb)
  x <- backsolve(R, y)
  x
}

x_chol <- ls_chol(A, b)
cat("Cholesky(NE) 解:", round(x_chol, 6), "\n")
Cholesky(NE) 解: -3.424658 2.993151

7.2 方法 B：QR（默认首选，数值稳定）

你不需要构造完整的 \(Q\)。核心是：把问题变成上三角回代。

在 R 中直接用：

  • qr.solve(A, b)（最小二乘解）
  • lm(y ~ x)（内部就是 QR）
展开代码 
x_qr <- qr.solve(A, b)
cat("QR 解:", round(x_qr, 6), "\n")
QR 解: -3.424658 2.993151
展开代码 
fit <- lm(price ~ area)
cat("lm() 系数:", round(coef(fit), 6), "\n")

7.3 方法 C：SVD（最稳；秩亏/近秩亏的“保险方案”）

适合：列（几乎）线性相关、奇异值尾部接近 0、你需要最小范数解。

展开代码 
ls_svd <- function(A, b, tol = 1e-12) {
  sv <- svd(A)
  d <- sv$d
  keep <- d > tol * max(d)
  Utb <- drop(t(sv$u) %*% b)
  coeff <- rep(0, length(d))
  coeff[keep] <- Utb[keep] / d[keep]
  x <- sv$v %*% coeff
  drop(x)
}

x_svd <- ls_svd(A, b)
cat("SVD 解:", round(x_svd, 6), "\n")
SVD 解: -3.424658 2.993151

8 方法选择：一课版“决策树”

把这 4 行当作你的实际工作流程（不需要背大定理）：

  1. 默认：qr.solve(A,b)（稳定、够快）
  2. 若 kappa2(A) 很大或你怀疑共线：看奇异值 svd(A)$d
  3. 若尾部奇异值接近 0（近秩亏）：用 SVD（或正则化）
  4. 若 \(\kappa(A)\) 很小且你要反复解很多次：可考虑 Cholesky（NE） 提速

9 综合案例 1：高阶多项式拟合为什么会不稳定？如何修复？

本案例的学习点不是“过拟合”本身，而是：
Vandermonde 矩阵会迅速病态 \(\Rightarrow\) 需要换基/缩放/正则化。

9.1 现象：Vandermonde 条件数随阶数飙升

x_fine <- seq(0, 1, length.out = 200)

kappa_by_degree <- function(x, max_deg = 18) {
  sapply(1:max_deg, function(d) {
    A <- outer(x, 0:d, "^")
    kappa2(A)
  })
}

kappas <- kappa_by_degree(x_data, 18)

这就是数值线代的思维：先改问题表示，再谈算法。

10 综合案例 2：共线性（近秩亏）与岭回归：从 SVD 视角看“被抑制的方向”

岭回归相当于在这些方向上“刹车”，让解不要乱飙。

10.1 岭回归与“滤波因子”可视化（建立直觉）

岭回归解： \[ \hat\beta_\lambda = (X^TX+\lambda I)^{-1}X^Ty. \] 在 SVD 视角下，小奇异值方向会被更强地抑制。

展开代码 
lambdas <- 10^seq(-6, 2, length.out = 200)
d <- svX$d

# 每个奇异值方向的“增益”（越小越抑制）
gain <- sapply(lambdas, function(lam) d / (d^2 + lam))  # proportional filter

12 总结：一课版你需要带走的“工作记忆”

12.1 1. 直觉

  • 最小二乘：在所有 \(\mathbf{Ax}\) 里找离 \(\mathbf{b}\) 最近的那个
  • 几何：\(\mathbf{A}\hat{\mathbf{x}}\) 是 \(\mathbf{b}\) 在 \(\text{col}(A)\) 的正交投影
  • 判据：\(\mathbf{A}^T(\mathbf{b}-\mathbf{A}\hat{\mathbf{x}})=0\)

12.2 2. 算法（按优先级）

  1. QR： qr.solve(A,b)（默认首选，稳）
  2. SVD： 近秩亏/很病态时的保险：看 svd(A)$d，必要时截断小奇异值
  3. Cholesky（NE）： 良态 + 多次求解 + 追求速度可用，但要记住风险：\(\kappa\) 平方化

12.3 3. 数值线代的核心习惯

  • 先问：问题是不是病态？（kappa2(A)、奇异值尾部）
  • 再选：合适的表示（正交基/缩放/正则化）与算法（QR/SVD）
  • 最后做：用残差、正交性检验等可观察量验证结果是否可信
