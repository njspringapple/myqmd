Voraussetzungen im Überblick

1.1 梯度：指南针

Gradient: Der Kompass

对于函数 \(f: \mathbb{R}^n \to \mathbb{R}\)，梯度 \(\nabla f(\mathbf{x})\) 是一个 \(n\) 维向量，指向函数值增长最快的方向。在每个点，它就像一个指南针，告诉你”往哪走会上坡最快”。

\[ \nabla f(\mathbf{x}) = \left(\frac{\partial f}{\partial x_1},\; \frac{\partial f}{\partial x_2},\;\ldots,\;\frac{\partial f}{\partial x_n}\right) \]

反过来，负梯度 \(-\nabla f(\mathbf{x})\) 指向”下坡最快”的方向——这正是梯度下降法的核心思想。

1.2 黑塞矩阵：地形的曲率

Hesse-Matrix: Die Krümmung des Geländes

黑塞矩阵 \(\mathbf{H}_f(\mathbf{x})\) 是二阶偏导数构成的 \(n \times n\) 对称矩阵。如果说梯度告诉你”坡度有多陡”，那么黑塞矩阵告诉你”坡度变化得有多快”——也就是地形的曲率。

\[ [\mathbf{H}_f(\mathbf{x})]_{ij} = \frac{\partial^2 f}{\partial x_i \,\partial x_j} \]

1.3 凸性：为什么它如此重要

Konvexität: Warum sie so wichtig ist

一个函数是凸的，直觉上意味着它的图形像一个”碗”——任意两点之间的连线始终在函数曲面之上。

\[ f\bigl(\lambda\mathbf{x} + (1-\lambda)\mathbf{y}\bigr) \;\le\; \lambda\,f(\mathbf{x}) + (1-\lambda)\,f(\mathbf{y}), \quad \forall\,\lambda \in [0,1] \]

凸函数的每个局部极小值都是全局极小值——这意味着任何”下山”方法最终都会到达碗底。

Bei konvexen Funktionen ist jedes lokale Minimum auch ein globales. Bei nicht-konvexen Funktionen kann man in lokalen Minima „stecken bleiben”.

2.1 从优化到方程求解

Von der Optimierung zum Gleichungslösen

优化问题和方程求解有着深刻的联系：要找函数 \(f\) 的极小值，若 \(f\) 可微，则在极小值处必有 \(\nabla f(\mathbf{x}^\star) = \mathbf{0}^\top\)。也就是说，求极小值 = 解方程 \(\nabla f = \mathbf{0}\)。因此我们先从方程求解讲起。

Optimierung und Gleichungslösen sind tief miteinander verbunden: Ein Minimum finden = die Gleichung \(\nabla f = \mathbf{0}\) lösen.

2.2 二分法：最朴素的策略

Bisektionsverfahren: Die einfachste Strategie

直觉：想象你在玩”猜数字”游戏——每次猜中间值，根据”大了”或”小了”缩小范围。

给定连续函数 \(f: \mathbb{R} \to \mathbb{R}\)，若 \(f(a) \cdot f(b) < 0\)（即两端异号），则由介值定理知 \([a, b]\) 之间必有零点。

# 二分法实现 | Bisektionsverfahren
二分法 <- function(f, a, b, eps = 1e-8, max_iter = 100) {
  history <- data.frame(iter = integer(), a = numeric(), b = numeric(), 
                        mid = numeric(), f_mid = numeric())
  for (k in 1:max_iter) {
    mid <- (a + b) / 2
    f_mid <- f(mid)
    history <- rbind(history, data.frame(iter = k, a = a, b = b, 
                                          mid = mid, f_mid = f_mid))
    if (b - a < eps) break
    if (f(a) * f_mid < 0) b <- mid else a <- mid
  }
  list(root = mid, history = history)
}

2.3 牛顿-拉弗森法：用切线逼近

Newton-Raphson-Verfahren: Annäherung durch Tangenten

直觉：二分法只知道”零点在左还是在右”，而牛顿法更聪明——它在当前点画切线，用切线与 \(x\) 轴的交点作为下一步的猜测。

对 \(f(x)\) 在 \(x^{(k)}\) 处作一阶泰勒展开：

\[ f(x) \approx f(x^{(k)}) + f'(x^{(k)})(x - x^{(k)}) \overset{!}{=} 0 \]

解出下一步：

\[ \boxed{x^{(k+1)} = x^{(k)} - \frac{f(x^{(k)})}{f'(x^{(k)})}} \]

几何意义：取函数在当前点切线与 \(x\) 轴的交点。

# 牛顿-拉弗森法 | Newton-Raphson-Verfahren
牛顿法_方程 <- function(f, f_deriv, x0, eps = 1e-10, max_iter = 50) {
  x <- x0
  history <- data.frame(iter = 0, x = x, fx = f(x))
  for (k in 1:max_iter) {
    fx <- f(x)
    fpx <- f_deriv(x)
    if (abs(fpx) < 1e-15) { warning("导数接近零"); break }
    x_new <- x - fx / fpx
    history <- rbind(history, data.frame(iter = k, x = x_new, fx = f(x_new)))
    if (abs(x_new - x) < eps) break
    x <- x_new
  }
  list(root = x, history = history)
}

Tip📌 二次收敛的含义 | Quadratische Konvergenz

牛顿法在解的附近具有二次收敛：每步迭代后，误差大约是上一步误差的平方。也就是说，如果某步误差是 \(10^{-3}\)，下一步大约是 \(10^{-6}\)，再下一步约 \(10^{-12}\)——有效数字每步翻倍！

Quadratische Konvergenz: Der Fehler wird in jedem Schritt etwa quadriert. Die Anzahl korrekter Stellen verdoppelt sich pro Schritt!

2.4 推广到多维：不动点迭代

Verallgemeinerung auf mehrere Dimensionen: Fixpunktiteration

对于多维方程 \(\mathbf{f}(\mathbf{x}) = \mathbf{0}\)（其中 \(\mathbf{f}: \mathbb{R}^n \to \mathbb{R}^n\)），一阶不动点迭代具有统一形式：

\[ \mathbf{x}^{(k)} = \mathbf{x}^{(k-1)} - \gamma\,\mathbf{f}(\mathbf{x}^{(k-1)}) \]

当 \(\rho := \|\mathbf{I}_n - \gamma\,\mathbf{J}_f(\mathbf{x}^\star)\| < 1\) 时，以线性速率收敛：\(\|\mathbf{x}^{(k)} - \mathbf{x}^\star\| = O(\rho^k)\)。

步长 \(\gamma\) 的选择至关重要：太小则收敛龟速，太大则根本不收敛。这个主题在后面的梯度下降中会更加深入地展开。

Important💡 与优化的联系 | Verbindung zur Optimierung

对 \(\min f(\mathbf{x})\) 应用牛顿法 = 对 \(\nabla f(\mathbf{x}) = \mathbf{0}\) 应用牛顿-拉弗森法！

牛顿法的迭代更新如下（把 \(\mathbf{f}\) 换成 \(\nabla f^\top\)，雅可比 \(\mathbf{J}_f\) 就变成黑塞 \(\mathbf{H}_f\)）：

\[ \mathbf{x}^{(k+1)} = \mathbf{x}^{(k)} - \mathbf{H}_f(\mathbf{x}^{(k)})^{-1}\,\nabla f(\mathbf{x}^{(k)})^\top \]

Newton-Verfahren für \(\min f\) = Newton-Raphson angewendet auf \(\nabla f = \mathbf{0}\).

3.1 为什么需要优化？

Warum brauchen wir Optimierung?

统计学与机器学习的核心问题几乎都可以表述为优化问题。极大似然估计（MLE）寻找最可能产生观测数据的参数：\(\hat{\boldsymbol{\theta}}_{\text{ML}} = \arg\max_{\boldsymbol{\theta}} \sum_{i=1}^n \log f_Y(y_i \mid \boldsymbol{\theta})\)。监督学习最小化经验损失：\(\hat{\boldsymbol{\theta}} = \arg\min_{\boldsymbol{\theta}} \sum_{i=1}^n L(y_i,\, p_{\boldsymbol{\theta}}(\mathbf{x}_i))\)。线性回归最小化残差平方和：\(\hat{\boldsymbol{\beta}} = \arg\min_{\boldsymbol{\beta}} \|\mathbf{y} - \mathbf{X}\boldsymbol{\beta}\|_2^2\)。正则化回归（岭回归或 Lasso）在损失中加入惩罚项：\(\hat{\boldsymbol{\beta}} = \arg\min_{\boldsymbol{\beta}} \|\mathbf{y} - \mathbf{X}\boldsymbol{\beta}\|_2^2 + \lambda\|\boldsymbol{\beta}\|_p\)。

上述正则化的惩罚形式等价于有约束形式：\(\arg\min_{\boldsymbol{\beta}} \|\mathbf{y} - \mathbf{X}\boldsymbol{\beta}\|_2^2\) 使得 \(\|\boldsymbol{\beta}\|_p \le c\)。

所有这些问题通常都无法直接求得解析解，需要迭代方法。

Nelder-Mead 的四种操作：在每一步中，找到单纯形中函数值最大（最差）的那个顶点，然后尝试用以下策略替换它——反射（Reflection）将最差点关于其余点的重心进行镜像翻转；扩张（Expansion）沿反射方向走得更远（如果反射效果好的话）；收缩（Contraction）在反射效果不好时向内收缩；压缩（Shrink）在所有策略都失败时将整个单纯形向最优点收缩。

4.3 梯度下降法

Gradientenabstiegsverfahren

核心思想：梯度指向上升方向 \(\Rightarrow\) 负梯度指向下降方向——沿负梯度走，函数值一定减小（步长足够小时）。

\[ \boxed{\mathbf{x}^{(k+1)} = \mathbf{x}^{(k)} - \gamma\,\nabla f(\mathbf{x}^{(k)})^\top} \]

其中 \(\gamma > 0\) 为步长（学习率）。

# 梯度下降核心实现 | Kern des Gradientenabstiegs
梯度下降 <- function(grad_f, x0, gamma, max_iter, f = NULL, tol = 1e-10) {
  x <- x0
  n <- length(x0)
  history <- matrix(NA, nrow = max_iter + 1, ncol = n)
  loss_hist <- numeric(max_iter + 1)
  history[1, ] <- x
  if (!is.null(f)) loss_hist[1] <- f(x)
  
  for (k in 1:max_iter) {
    g <- grad_f(x)
    x_new <- x - gamma * g
    history[k + 1, ] <- x_new
    if (!is.null(f)) loss_hist[k + 1] <- f(x_new)
    if (sqrt(sum((x_new - x)^2)) < tol) {
      history <- history[1:(k+1), , drop = FALSE]
      loss_hist <- loss_hist[1:(k+1)]
      break
    }
    x <- x_new
  }
  list(x = x, history = history, loss = loss_hist, 
       iters = nrow(history) - 1)
}

4.4 梯度下降的收敛性

Konvergenz des Gradientenabstiegs

梯度下降的收敛速度取决于函数的性质。核心概念是梯度的 Lipschitz 连续性：若 \(\|\nabla f(\mathbf{x}) - \nabla f(\mathbf{y})\| \le L\|\mathbf{x} - \mathbf{y}\|\) 对所有 \(\mathbf{x}, \mathbf{y}\) 成立，则称梯度以常数 \(L\) Lipschitz 连续。这等价于黑塞矩阵的谱范数有界：\(\|\mathbf{H}_f(\mathbf{x})\| \le L\)——也就是曲率不会无限大。

对于一般凸函数，取 \(\gamma \le \frac{1}{L}\)，收敛速率为：

\[ f(\mathbf{x}^{(k)}) - f(\mathbf{x}^\star) \le \frac{\|\mathbf{x}^{(0)} - \mathbf{x}^\star\|^2}{2\gamma k} \]

误差以 \(O(1/k)\) 的速率下降——这意味着要将误差减小到 \(\epsilon\)，需要约 \(k \sim 1/\epsilon\) 步（较慢）。

对于强凸函数（即 \(\mathbf{H}_f(\mathbf{x}) \succeq \mu\mathbf{I}\) 对某个 \(\mu > 0\) 成立），收敛速率大幅改善：

\[ f(\mathbf{x}^{(k)}) - f(\mathbf{x}^\star) \le \rho^k \cdot \bigl(f(\mathbf{x}^{(0)}) - f(\mathbf{x}^\star)\bigr), \quad \rho = 1 - \frac{\mu}{L} < 1 \]

这是指数收敛（线性收敛率），仅需 \(k \sim \log(1/\epsilon)\) 步！

条件数 \(\kappa_f = \frac{L}{\mu} = \frac{\lambda_{\max}(\mathbf{H}_f)}{\lambda_{\min}(\mathbf{H}_f)}\) 决定了收敛快慢：\(\kappa\) 越大（椭圆越扁），收敛越慢。

4.5 停止准则

Abbruchkriterien

何时停止迭代？通常组合使用以下准则：梯度足够小（\(\|\nabla f(\mathbf{x}^{(k)})\| < \varepsilon_{\text{grad}}\)）、函数值不再明显变化（\(|f(\mathbf{x}^{(k+1)}) - f(\mathbf{x}^{(k)})| / |f(\mathbf{x}^{(k)})| < \varepsilon_f\)）、或超过最大迭代次数（\(k > k_{\max}\)）。典型取值为 \(\varepsilon_{\text{grad}} \approx 10^{-6}\) 至 \(10^{-8}\)，\(k_{\max} \approx 1000\) 至 \(10000\)。

4.6 直线搜索：自动调节步长

Liniensuche: Automatische Schrittweitensteuerung

固定步长的梯度下降需要手动调参。回溯直线搜索（Armijo 条件） 能自动选择合适的步长。

给定下降方向 \(\mathbf{d} = -\nabla f(\mathbf{x}^{(k)})^\top\)，参数 \(c \in (0,1)\) 和 \(\rho \in (0,1)\)，算法的流程是：初始令 \(\gamma = 1\)，然后不断检查”下降是否足够”——如果 \(f(\mathbf{x}^{(k)} + \gamma\mathbf{d}) > f(\mathbf{x}^{(k)}) + c\gamma\nabla f(\mathbf{x}^{(k)})\mathbf{d}\)，就将 \(\gamma \leftarrow \rho\gamma\) 缩小步长；否则接受当前步长。

# 回溯直线搜索 | Backtracking-Liniensuche
回溯直线搜索 <- function(f, grad_f, x, d, c1 = 1e-4, rho = 0.5) {
  gamma <- 1.0
  fx <- f(x)
  gd <- sum(grad_f(x) * d)  # 方向导数
  while (f(x + gamma * d) > fx + c1 * gamma * gd) {
    gamma <- gamma * rho
  }
  return(gamma)
}

Armijo 条件的直觉是：我们不要求步长完美地最小化沿搜索方向的函数值，只要求它带来”足够的下降”。典型参数：\(c = 10^{-4}\)，\(\rho = 0.5\)。

4.7 牛顿法：利用曲率加速

Newton-Verfahren: Beschleunigung durch Krümmungsinformation

梯度下降只看”坡度”（一阶），而牛顿法还看”坡度变化得有多快”（二阶）——它用二阶泰勒展开的抛物面来局部近似函数。

\[ f(\mathbf{x} + \mathbf{h}) \approx f(\mathbf{x}) + \nabla f(\mathbf{x})\,\mathbf{h} + \frac{1}{2}\mathbf{h}^\top\mathbf{H}_f(\mathbf{x})\,\mathbf{h} \]

对 \(\mathbf{h}\) 求极小，令其梯度为零：\(\nabla f(\mathbf{x})^\top + \mathbf{H}_f(\mathbf{x})\,\mathbf{h} = \mathbf{0}\)，解出牛顿步：

\[ \boxed{\mathbf{x}^{(k+1)} = \mathbf{x}^{(k)} - \mathbf{H}_f(\mathbf{x}^{(k)})^{-1}\,\nabla f(\mathbf{x}^{(k)})^\top} \]

Warning⚠️ 实践要点 | Praktischer Hinweis

永远不要直接计算矩阵的逆 \(\mathbf{H}_f^{-1}\)！应该求解线性方程组 \(\mathbf{H}_f\,\mathbf{h} = -\nabla f^\top\)。

Berechnen Sie niemals explizit die Inverse \(\mathbf{H}_f^{-1}\)! Lösen Sie stattdessen das lineare Gleichungssystem.

  x <- x0
  history <- matrix(NA, nrow = max_iter + 1, ncol = length(x0))
  history[1, ] <- x
  for (k in 1:max_iter) {
    g <- grad_f(x)
    H <- hess_f(x)
    h <- -solve(H, g)  # 解线性方程组，而非求逆！
    x_new <- x + h
    history[k + 1, ] <- x_new
    if (sqrt(sum(h^2)) < tol) {
      history <- history[1:(k+1), , drop = FALSE]
      break
    }
    x <- x_new
  }
  list(x = x, history = history[complete.cases(history), , drop = FALSE])
}

4.8 拟牛顿法与 BFGS

Quasi-Newton-Verfahren und BFGS

问题：牛顿法太贵——黑塞矩阵计算 \(O(n^2)\) 存储、\(O(n^3)\) 求解。

思路：不算黑塞矩阵，而是从梯度的变化中迭代近似黑塞逆矩阵 \(\mathbf{H}_f^{-1}\)。

BFGS 更新公式：令 \(\mathbf{s}_k = \mathbf{x}^{(k+1)} - \mathbf{x}^{(k)}\)，\(\mathbf{y}_k = (\nabla f(\mathbf{x}^{(k+1)}) - \nabla f(\mathbf{x}^{(k)}))^\top\)，\(\rho_k = \frac{1}{\mathbf{y}_k^\top\mathbf{s}_k}\)：

\[ \mathbf{B}_{k+1} = (\mathbf{I} - \rho_k\,\mathbf{s}_k\mathbf{y}_k^\top)\,\mathbf{B}_k\,(\mathbf{I} - \rho_k\,\mathbf{y}_k\mathbf{s}_k^\top) + \rho_k\,\mathbf{s}_k\mathbf{s}_k^\top \]

其中 \(\mathbf{B}_k \approx \mathbf{H}_f^{-1}(\mathbf{x}^{(k)})\)。

BFGS 每步复杂度为 \(O(n^2)\)（牛顿法 \(O(n^3)\)），具有超线性收敛，接近牛顿法的速度。对于更大规模的问题，L-BFGS（有限内存版）仅存储最近 \(m\) 个向量，内存复杂度降至 \(O(mn)\)。

BFGS 是 R 中 optim() 提供梯度时的默认方法，也是统计推断中最常用的优化器。

4.9 动量梯度下降

Gradientenabstieg mit Impuls (Momentum)

动机：在条件数 \(\kappa\) 很大的问题上，梯度下降频繁”锯齿形”震荡。

类比：想象一个球从山坡滚下——它不会在每个小坑里停住，因为惯性（“Schwung”）会带它冲过去。

\[ \mathbf{v}^{(k+1)} = \alpha\,\mathbf{v}^{(k)} - \gamma\,\nabla f(\mathbf{x}^{(k)})^\top \] \[ \mathbf{x}^{(k+1)} = \mathbf{x}^{(k)} + \mathbf{v}^{(k+1)} \]

其中 \(\alpha \in [0, 1)\) 为阻尼系数（通常取 \(0.9\)），\(\gamma\) 为步长。

# 动量梯度下降实现 | Momentum-GD
动量梯度下降 <- function(grad_f, x0, gamma, alpha, max_iter, f = NULL) {
  x <- x0; v <- rep(0, length(x0))
  n <- length(x0)
  history <- matrix(NA, nrow = max_iter + 1, ncol = n)
  loss_hist <- numeric(max_iter + 1)
  history[1, ] <- x
  if (!is.null(f)) loss_hist[1] <- f(x)
  
  for (k in 1:max_iter) {
    g <- grad_f(x)
    v <- alpha * v - gamma * g
    x <- x + v
    history[k + 1, ] <- x
    if (!is.null(f)) loss_hist[k + 1] <- f(x)
  }
  list(x = x, history = history, loss = loss_hist)
}

4.10 随机梯度下降（SGD）

Stochastischer Gradientenabstieg (SGD)

动机：典型的机器学习问题需要最小化经验风险 \(R(\boldsymbol{\theta}) = \frac{1}{N}\sum_{i=1}^N L(y_i,\, p_{\boldsymbol{\theta}}(\mathbf{x}_i))\)。全梯度每步需要遍历全部 \(N\) 个样本，当 \(N\) 达到百万甚至十亿级时——一步都算不完。

关键洞察：单个样本的梯度是全梯度的无偏估计：

\[ \mathbb{E}\bigl[\nabla L(y_i,\, p_{\boldsymbol{\theta}}(\mathbf{x}_i))\bigr] = \nabla R(\boldsymbol{\theta}) \]

这意味着我们可以用一个或一小批样本来”代替”全体样本估计梯度方向——方向”平均而言”是对的，只是带有噪声。

SGD 更新（随机选取索引 \(i_k\)）：

\[ \boldsymbol{\theta}^{(k+1)} = \boldsymbol{\theta}^{(k)} - \gamma^{(k)}\,\nabla L(y_{i_k},\, p_{\boldsymbol{\theta}}(\mathbf{x}_{i_k}))^\top \]

每步代价从 \(O(N)\) 降为 \(O(1)\)（或 Mini-batch 的 \(O(B)\)）！

5.1 Lagrange 乘数法

Lagrange-Multiplikator-Methode

问题：最小化 \(f(\mathbf{x})\)，同时满足等式约束 \(g_i(\mathbf{x}) = 0\)，\(i = 1, \ldots, m\)。

几何直觉：在约束面上的最优点，\(f\) 的等高线与约束面相切——也就是 \(\nabla f\) 平行于 \(\nabla g\)。如果 \(\nabla f\) 不平行于 \(\nabla g\)，你就还能沿约束面移动来降低 \(f\)。

\[ \exists\,\lambda \quad\text{使得}\quad \nabla f(\mathbf{x}^\star) + \lambda\,\nabla g(\mathbf{x}^\star) = \mathbf{0} \]

Lagrange 函数：将约束”嵌入”目标函数——

\[ \mathcal{L}(\mathbf{x}, \boldsymbol{\lambda}) = f(\mathbf{x}) + \boldsymbol{\lambda}^\top \mathbf{g}(\mathbf{x}) \]

在最优点处的必要条件：\(\nabla_{\mathbf{x}} \mathcal{L}(\mathbf{x}^\star, \boldsymbol{\lambda}^\star) = \mathbf{0}^\top\) 且 \(\mathbf{g}(\mathbf{x}^\star) = \mathbf{0}\)。

5.2 KKT 条件：处理不等式约束

KKT-Bedingungen: Ungleichungsnebenbedingungen

当问题包含不等式约束 \(h_j(\mathbf{x}) \le 0\) 时，需要更强的条件。Karush-Kuhn-Tucker（KKT）条件是有约束优化最优性的必要条件（对凸问题同时也是充分条件）。

在最优点 \(\mathbf{x}^\star\) 处，必须同时满足四个条件：

\[ \text{(1) 驻定性: } \nabla f(\mathbf{x}^\star) + \sum_i \lambda_i\,\nabla g_i(\mathbf{x}^\star) + \sum_j \mu_j\,\nabla h_j(\mathbf{x}^\star) = \mathbf{0}^\top \] \[ \text{(2) 原始可行性: } g_i(\mathbf{x}^\star) = 0, \quad h_j(\mathbf{x}^\star) \le 0 \] \[ \text{(3) 对偶可行性: } \mu_j \ge 0 \] \[ \text{(4) 互补松弛性: } \mu_j\,h_j(\mathbf{x}^\star) = 0 \quad \forall\, j \]

Tip🤔 互补松弛条件的直觉 | Intuition der komplementären Schlupfbedingung

条件 \(\mu_j\,h_j(\mathbf{x}^\star) = 0\) 意味着：对每个不等式约束，二者必居其一——

  • 如果 \(h_j(\mathbf{x}^\star) < 0\)（约束”不紧”，有余量），则 \(\mu_j = 0\)（这个约束”不起作用”）
  • 如果 \(\mu_j > 0\)（这个约束在”推”最优解），则 \(h_j(\mathbf{x}^\star) = 0\)（约束恰好取等号）

直觉：只有真正”挡住路”的围墙才会对你的选择产生影响。

Nur Wände, die tatsächlich „im Weg stehen”, beeinflussen die optimale Lösung.

Important💡 KKT 与凸性 | KKT und Konvexität

定理：若 \(f\) 和所有 \(h_j\) 是凸函数，所有 \(g_i\) 是仿射函数，则 KKT 条件不仅是必要条件，也是充分条件——满足 KKT 的点就是全局最优。

许多统计问题是凸的（最小二乘、指数族 MLE、岭回归等），所以 KKT 在统计学中特别有力。对非凸问题（如神经网络），KKT 仅给出局部最优的保证。
